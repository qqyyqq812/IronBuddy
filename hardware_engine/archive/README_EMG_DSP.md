# EMG 模拟肌电信号 DSP 处理计算逻辑

> 供 Worker 2 与硬件端 (Worker 1) 数据对齐及下游特征提取使用的数字信号处理核心逻辑。

## 1. 信号形态特征认知
模拟肌电 (sEMG) 传感器采撷到的原始信号通常具备以下痛点：
- **毛刺极多**：包含高频随机肌电噪声及环境白噪声。
- **基线漂移 (Baseline Wander)**：由受试者呼吸、肢体运动和电极位移造成极低频（<10Hz）漂移。
- **工频干扰**：电网环境固有的 50Hz/60Hz 强干扰。
- **双极性**：信号在基准电平（如 ADC 的中间值，例如 ESP32 的 1.65V）上下震荡，均值趋于零，无法直接代表“发力大小”。

## 2. 算力约束与核心算法链路
基于目标设备的算力约束，以及需要将原始数据转换为 `0~100%` 平滑权重（与 NPU 骨骼视觉数据做视感融合）的业务需求，选用如下**时域核心处理流 (DSP Pipeline)**：

### Step A: 零点偏移校准 (DC Offset Removal)
- **动作**：通过启动阶段前 N 个采样点求均值，或使用极低截止频率的高通滤波器（如 `fc = 20Hz`），消除信号基线漂移。
- **公式**：`x_centered(t) = x(t) - mean(x(t-N...t))`

### Step B: 陷波抗干扰 (Notch Filtering)
- **动作**：滤除 `50Hz` (或 `60Hz`) 强频干扰。
- **简易平替方案**：若想避免设计 IIR 陷波器，也可采用滑动平均窗口大小等于 $1/50\text{Hz} = 20\text{ms}$ 宽度的整数倍，在后续环节自然隐式抵消工频。

### Step C: 全波整流 (Full-wave Rectification)
- **动作**：将去直流后的信号取绝对值，翻转负半周期的波形。
- **公式**：`x_rectified(t) = |x_centered(t)|`

### Step D: 包络提取 (Envelope Extraction via RMS / LPF)
- **动作**：获取能够表征“肌肉激活收缩力度”的低频包络。这里推荐**滑动均方根 (Moving RMS)** 或简单的**滑动平均/低通滤波 (Moving Average / LPF)**。
- 考虑到计算效率与实时跟手感，选用 **滑动均方根 (Moving RMS)**，兼顾能量特性的捕捉：
- **公式**：`RMS(t) = sqrt( mean( x_centered^2 ) )` （窗口时长取 `50ms~150ms`）

### Step E: 最大自主收缩 (MVC) 零-百归一化
- **动作**：收集静息基底噪声 `RMS_min` 和最大肌肉爆发阈值 `RMS_max`。
- **公式**：`Activation_Level = clamp((RMS(t) - RMS_min) / (RMS_max - RMS_min), 0.0, 1.0) * 100%`

## 3. 并行工程接口建议
为给 Worker 3 渲染“腿部肌肉群发光热力材质”，本 Pipeline 最终将高频迭代输出单个归一化浮点/整数，并写入状态机 `/dev/shm/fsm_state.json`。
