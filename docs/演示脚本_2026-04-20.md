## Step 1 · 展示"数据库三位一体"（30 秒）

**📍 终端 A**：
```bash
cd /home/qq/projects/embedded-fullstack
python3 scripts/demo_view.py overview
```

**👀 看屏幕上这 3 行**：
- `三天训练 seed=3 场 / 143 reps` ← 客观训练数据
- `voice_sessions ≥9 条，4/16、4/19、4/20 各一次"膝盖"` ← 主观闲聊
- `preference_history 5+ 条演化` ← 学出来的偏好

**💬 讲**："注意时间线——4/16 第一次提膝盖疼，4/19 深蹲后复发，4/20 再提第三次。系统自己从闲聊里听出来，用户不用打卡。"

## Step 2 · 手动催一次偏好学习（60 秒）

**📍 终端 A**：
```bash
touch /dev/shm/openclaw_trigger_preference_learning
```

**👀 转去看终端 B**（最多等 60 秒），会出现：
```
[trigger] 检测到 preference_learning 手动触发文件
[preference_learning] 规则引擎模式启动
[preference_learning] daily_summary 写入: ... rec=膝盖多次提及，次日避免下肢高强度
[preference_learning] new prompt version=X changed_fields=[...]
```

**💬 讲**："daemon 每 60 秒扫一次触发点。这条 `rec` 是规则引擎的产物——稳定可复现，生产设 `OPENCLAW_PREFLEARN_MODE=llm` 就切 DeepSeek 做个性化推导。"

## 常见问题速答

| 老师可能问 | 一句话答 |
|---|---|
| 数据真的吗？ | 4/16-4/20 是演示种子（`is_demo_seed=1`），真实数据走同一 schema。v7-v9 是 live 产出。 |
| AI 是谁？ | 演示用规则引擎保稳定；生产切 DeepSeek（`OPENCLAW_PREFLEARN_MODE=llm`）。 |
| 飞书为啥没发？ | 护栏，`IRONBUDDY_FEISHU_DRY_RUN=1`。生产关掉即真发。 |
| 存录音吗？ | 不存。`voice_sessions` 只存转写文本 + 回复。 |
| 怎么部署板端？ | `openclaw_daemon` 已 systemd-ready，并入 `start_all_services.sh` 即可。 |
