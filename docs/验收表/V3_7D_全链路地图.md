### §3.3 采集 → 存储路径映射 ⚠️

读 [`scripts/collect_one.sh`](../../scripts/collect_one.sh) 第 9、53、54 行源码：

```bash
SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
BOARD_OUT_DIR="/home/toybrick/training_data/${EXERCISE}/${LABEL}"
LOCAL_OUT_DIR="${SCRIPT_DIR}/data/${EXERCISE}/${LABEL}"
```

因为 `SCRIPT_DIR = /home/qq/projects/embedded-fullstack/scripts`，所以 `LOCAL_OUT_DIR` 实际展开成：

```
/home/qq/projects/embedded-fullstack/scripts/data/<ex>/<label>/
```

**结论**：周一新采数据会落在 `scripts/data/bicep_curl/<label>/`，**不是** `data/bicep_curl/<label>/`。

**建议**：训练时要么指定 `--data scripts/data/bicep_curl`，要么先 `mv scripts/data/bicep_curl/* data/bicep_curl/` 合并。

### §3.4 CSV 10 列 Schema

| 列 | 单位 / 范围 | 含义 | 来源 |
|----|-------------|------|------|
| 1. `Timestamp` | 秒 (epoch) | 采样时刻 | `time.time()` |
| 2. `Ang_Vel` | 度/s | 关节角速度 | diff(Angle)/dt |
| 3. `Angle` | 度 [0°,180°] | 三点法关节角 | shoulder-elbow-wrist（弯举）/ hip-knee-ankle（深蹲） |
| 4. `Ang_Accel` | 度/s² | 关节角加速度 | diff(Ang_Vel)/dt |
| 5. `Target_RMS` | 0~1 归一化 | 目标肌 RMS（弯举=肱二头） | EMG 通道 A |
| 6. `Comp_RMS` | 0~1 | 代偿肌 RMS（弯举=前臂屈肌） | EMG 通道 B |
| 7. `Symmetry_Score` | 0~1 | 左右对称度 | RMS 左右比 |
| 8. `Phase_Progress` | 0~1 | 动作阶段进度（离心/向心） | FSM 相位 |
| 9. `pose_score` | 0~1 | 关键点置信度平均 | NPU / Cloud 输出 |
| 10. `label` | str | `golden` / `lazy` / `bad` | CLI `--mode` 注入 |

CSV header 来自 [`collect_training_data.py:54-60`](../../tools/collect_training_data.py)。

## §4 数据预处理（内嵌在采集）

预处理**没有**单独脚本，全部实现在 [`tools/collect_training_data.py`](../../tools/collect_training_data.py) 里。

### §4.1 三点法关节角

- 弯举：`shoulder-elbow-wrist` 三点计算肘关节角
- 深蹲：`hip-knee-ankle` 三点计算膝关节角
- 公式：`θ = acos((ba·bc)/(|ba|·|bc|))`
- 参考实现：`main_claw_loop.py:98-110` `SquatStateMachine.calculate_angle`（与采集脚本同源）

### §4.2 EMG RMS 与 `muscle_activation.json` 对接

```python
POSE_SHM        = "/dev/shm/pose_data.json"           # L41
EMG_SHM         = "/dev/shm/muscle_activation.json"   # L42
RECORD_MODE_SHM = "/dev/shm/record_mode"              # L43
```

由 `udp_emg_server.py` 原子写 JSON，采集脚本 20 Hz 轮询。

### §4.3 pose_score 阈值

```python
MIN_POSE_SCORE = 0.10   # L50
```

⚠️ **NPU 量化模型置信度普遍 0.1~0.2**（见 `.claude/rules/toybrick_board_rules.md` 第 3 条），不能按云端 0.5 卡。

### §4.4 时间一致性窗口

```python
MAX_TIMESTAMP_DIFF_S = 0.100   # L49：100 ms coherence window
```

`|t_pose − t_emg| > 0.1s` 的帧丢弃。

### §4.5 角度幅度过滤

```python
MIN_ANGLE_RANGE_DEG   = 20.0   # L51
WINDOW_SIZE_FOR_RANGE = 30     # L52（30 帧 = 1.5s @20Hz）
```

30 帧滑窗内角度振幅 < 20° 视为静止帧，剔除。

### §5.1 [`train_model.py`](../../tools/train_model.py) 参数

| 参数 | 默认 | 典型改法 |
|------|------|----------|
| `--data DIR` | - | 必填，指向 `data/bicep_curl` 或 `scripts/data/bicep_curl` |
| `--out DIR` | `models/` | 默认 OK |
| `--epochs` | 20 | 冒烟测试用 3；正式训 20-50 |
| `--batch` | 32 | CPU 训练建议 16 |
| `--lr` | 1e-3 | Adam 默认 |

```
     + 0.6 · CE(class_logits, class_gt)
```

3-head 结构：similarity（回归）+ class（3 分类 golden/lazy/bad）+ fatigue（辅助头）。

## §6 上板部署（含权重坑修复）⚠️

### §6.1 关键命令（每次重训后必做）

```bash
cp models/extreme_fusion_gru.pt hardware_engine/extreme_fusion_gru.pt
```

**为什么**：读 [`main_claw_loop.py:40-64`](../../hardware_engine/main_claw_loop.py) 的 `_load_gru_model()`：

```python
_dir = os.path.dirname(os.path.abspath(__file__))   # = hardware_engine/
candidates = [
    os.path.join(_dir, "extreme_fusion_gru.pt"),           # 首选
    os.path.join(_dir, "cognitive", "extreme_fusion_gru.pt"),  # 备用
]
```

- **首选** `hardware_engine/extreme_fusion_gru.pt`
- **备用** `hardware_engine/cognitive/extreme_fusion_gru.pt`
- `models/` 下的 `.pt` **不会**被 `_load_gru_model()` 找到

当前本地：`models/extreme_fusion_gru.pt` 存在，`hardware_engine/` 下**两个位置都没有** `.pt`。

**不做 §6.1 cp 操作 → 板端日志会出现 `[GRU] No model file found — inference disabled.` (L63)**

### §6.2 rsync via `start_validation.sh`

读 [`scripts/start_validation.sh:35-45`](../../scripts/start_validation.sh)：

```bash
rsync -az --checksum -e "ssh ..." \
    --exclude='.git' --exclude='*.tar.gz' --exclude='*.rar' \
    --exclude='docs/hardware_ref' --exclude='backups' --exclude='.agent_memory' \
    --exclude='data' \
    "$PROJECT_DIR/" $TARGET:/home/toybrick/streamer_v3/

# 兜底 scp（强制同步权重）
if [ -f "$PROJECT_DIR/models/extreme_fusion_gru.pt" ]; then
    scp "$PROJECT_DIR/models/extreme_fusion_gru.pt" \
        $TARGET:/home/toybrick/streamer_v3/hardware_engine/cognitive/extreme_fusion_gru.pt
fi
```

- rsync exclude 清单**不包含** `hardware_engine/` 或 `*.pt` → 本地 `hardware_engine/extreme_fusion_gru.pt` 会被正常同步到板端首选路径
- 兜底 scp 把 `models/` 下的 `.pt` 额外推到板端 `hardware_engine/cognitive/` 备用路径
- **所以**：只要本地做了 §6.1 的 cp，板端两个路径都会有权重，双保险

### §6.3 板端搜索顺序

```
/home/toybrick/streamer_v3/hardware_engine/extreme_fusion_gru.pt          ← 首选
/home/toybrick/streamer_v3/hardware_engine/cognitive/extreme_fusion_gru.pt ← 备用
```

### §6.4 推理触发

[`main_claw_loop.py:665-683`](../../hardware_engine/main_claw_loop.py)（DumbbellCurlFSM `rep-complete` 事件）：

```python
# 30 帧 7D 特征滑窗 → GRU forward
if len(_gru_feature_buf) >= _GRU_WINDOW_SIZE:
    features = np.array(_gru_feature_buf[-_GRU_WINDOW_SIZE:])  # [30, 7]
    out = _GRU_MODEL.forward(features)
    # out = {similarity, class_probs, fatigue}
```

每 `_GRU_INFER_EVERY=3` 帧推理一次（L35，节省 CPU）。

```bash
# → pkill -9 -f main_claw_loop && 重启 FSM 进程

# 拍深蹲镜头前
bash scripts/switch_model.sh squat
# → cp models/extreme_fusion_gru_squat.pt hardware_engine/extreme_fusion_gru.pt
# → pkill + 重启
```

注意：每次切换后第一个 rep 的分类结果可能冷启动偏差，建议热身 3 rep 再开录。

## §8 常见错误速查

### §8.1 `[GRU] No model file found — inference disabled.`

- **原因**：`hardware_engine/extreme_fusion_gru.pt` 和 `hardware_engine/cognitive/extreme_fusion_gru.pt` 都不存在
- **修**：`cp models/extreme_fusion_gru.pt hardware_engine/extreme_fusion_gru.pt`，然后重跑 `bash scripts/start_validation.sh`

### §8.2 `/api/muscle_activation` 返回空

- **原因**：`/dev/shm/emg_heartbeat` 不存在 → `udp_emg_server.py` 没起或 crash
- **修**：`ssh toybrick "ps aux | grep udp_emg"`，如无进程 → `bash scripts/stop_collect.sh && bash scripts/start_collect.sh`

### §8.3 采集 CSV 过少（60s 应 ~1200 行，实际 < 200 行）

- **原因 A**：摄像头 pose_score 过低（NPU 量化模型小动作置信度不足 0.1）
- **修 A**：确保动作幅度够大 + 距摄像头 1.5~2 m + 光线充足
- **原因 B**：EMG 没接通
- **修 B**：检查 `/dev/shm/muscle_activation.json` 是否有心跳更新（`ssh toybrick "cat /dev/shm/muscle_activation.json"`）

### §8.4 板端 i2c ES7243 -6 错误（`dmesg` 刷屏）

- **原因**：板载麦阵列硬件损坏（见 `decisions.md` 坑 10）
- **修**：`export VOICE_FORCE_MIC=plughw:2,0` 改用 USB 摄像头麦
- **延伸**：需同时改 `voice_daemon.py:356` fallback 链（当前不含 `plughw:2,0`，见 `语音debug步骤.md`）

### §8.5 rsync 超时

- **修**：先 `ssh toybrick "ls ~"` 测 SSH 通畅，再重试
- **备**：检查 `~/.ssh/id_rsa_toybrick` 权限 `chmod 600`
