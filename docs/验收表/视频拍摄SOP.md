# 视频拍摄 SOP（周一下午 5 min 演示视频）

> **版本**：shoot-v1 · **生成时间**：2026-04-18
> **目标**：5 min 视频覆盖 IronBuddy 全功能（UI/视觉/FSM/语音/EMG/GRU/自动教练/深蹲/飞书/统计）
> **假设**：板卡已上电 + 网络 + 云端 GPU 在线 + 权重已 cp 到 `hardware_engine/` + GRU 重训完成

---

## §1 拍摄前 3 小时准备

### 清单

- [ ] **板卡**：上电 + 摄像头接好 + 电极贴贴好（肱二头 + 前臂屈肌 2 片）
- [ ] **板端健康**：`ssh toybrick "uptime && df -h /"` 确认 CPU/磁盘正常
- [ ] **网络**：WiFi 稳定（测 `ping 10.105.245.224` < 50ms）
- [ ] **云端 GPU**：`curl http://localhost:6006/health` 返回 `"ready":true`
- [ ] **电池**：笔记本 ≥ 80%；板卡 USB 供电稳
- [ ] **道具**：5 kg 哑铃 × 1 / 电极贴 × 2 / 三脚架 / 补光灯 / USB 摄像头（做备用麦）
- [ ] **浏览器**：Chrome / Edge，全屏打开 `http://10.105.245.224:5000/`，关掉扩展
- [ ] **录屏工具**：OBS 预设 1920×1080 @30fps / 音频轨独立
- [ ] **权重**：本地做过 `cp models/extreme_fusion_gru.pt hardware_engine/extreme_fusion_gru.pt` ✅
- [ ] **GRU 重训**：`models/extreme_fusion_gru.pt` 的 `mtime` 是今天（`ls -la models/`）
- [ ] **服务**：`bash scripts/start_validation.sh` 已执行，5 服务就绪

## §3 NG 处理

- 每镜允许 **3 次 retake**
- 超出 3 次 → 走降级（见表右列）
- 特别规则：
  - 镜头 4（语音）NG ≥ 2 次 → 直接按钮降级，别反复试
  - 镜头 5（GRU）NG → 回放预采 CSV（周日晚提前录 1 段，存 `docs/验收表/fallback_demo.mp4`，不提交到 git）

## §7 异常应对

| 现象 | 应对 |
|------|------|
| 视频流黑屏 | 浏览器 F5；仍黑 → `bash scripts/restart_streamer.sh` |
| FSM 不计数 | 动作幅度加大 + 距离摄像头 1.5-2m + 补光灯 |
| EMG 波形平线 | `ssh toybrick "cat /dev/shm/muscle_activation.json"` 查心跳；无心跳 → 重启 `udp_emg_server.py` |
| GRU 浮窗不出 | 检查板端日志 `ssh toybrick "tail -50 /tmp/npu_main.log"` 搜 `[GRU]`；若 `No model file found` → §6.1 cp 没做 |
| 语音不响应 | 立即切按钮触发降级，字幕标注 |
