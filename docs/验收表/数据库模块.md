## 1. 模块定位与一句话架构

**目的**：让 IronBuddy 从"只会计数 + 短视闲聊"升级为"越用越懂你"的自进化健身教练。

**一句话闭环**：
> 训练数据 + 闲聊文本（真相源）→ OpenClaw daemon 定时吃数据（每日 23:00）→ 规则引擎产出偏好变化 → 自动生成新版 DeepSeek 系统提示词并切换 active → 下一次语音闲聊自动用新 prompt → 飞书 09:00 早报 / 周日 20:00 周报定点播报。

**两个 LLM 的分工**（严禁合并）：
- **DeepSeek（前端实时）**：语音闲聊 + 教练点评，延迟 ≤2s，短视无长历史注入，由 `voice_daemon.py` 直连 REST API。
- **OpenClaw（后端常驻）**：每日计划/周报/偏好学习，无延迟约束，读全量 `llm_log + voice_sessions + rep_events + daily_summary` 做批处理。走 `openclaw_daemon.py`，**演示期默认用规则引擎**（`OPENCLAW_PREFLEARN_MODE=rule`），生产可切 `llm`。
- **共享桥梁**：`data/ironbuddy.db`（SQLite WAL 模式，跨进程安全）。

### 3.1 训练类（3 张）

```sql
-- 一次训练会话（从开始到结束）
CREATE TABLE training_sessions (
  id              INTEGER PRIMARY KEY,
  started_at      TEXT NOT NULL,       -- ISO8601
  ended_at        TEXT,
  exercise        TEXT NOT NULL,       -- 'bicep_curl' / 'squat' / 'curl'
  good_count      INTEGER DEFAULT 0,
  failed_count    INTEGER DEFAULT 0,
  fatigue_peak    REAL DEFAULT 0,      -- EMG 疲劳累积峰值
  duration_sec    INTEGER DEFAULT 0,
  is_demo_seed    INTEGER DEFAULT 0    -- 1=演示种子，0=真实
);

-- 单次动作（一次深蹲/弯举）
CREATE TABLE rep_events (
  id              INTEGER PRIMARY KEY,
  session_id      INTEGER,             -- FK to training_sessions
  ts              TEXT NOT NULL,
  is_good         INTEGER NOT NULL,    -- 1=good, 0=bad
  angle_min       REAL,                -- 最小角度（深蹲底点 / 弯举顶点）
  emg_target      REAL,                -- 目标肌 RMS
  emg_comp        REAL,                -- 代偿肌 RMS
  is_demo_seed    INTEGER DEFAULT 0
);

-- 每日总结（daemon 23:00 批处理写入）
CREATE TABLE daily_summary (
  date            TEXT PRIMARY KEY,    -- YYYY-MM-DD
  session_count   INTEGER DEFAULT 0,
  total_good      INTEGER DEFAULT 0,
  total_failed    INTEGER DEFAULT 0,
  total_fatigue   REAL DEFAULT 0,
  best_streak     INTEGER DEFAULT 0,
  is_demo_seed    INTEGER DEFAULT 0,
  summary         TEXT,                -- 自然语言总结
  rec             TEXT                 -- 明日建议
);
```

### 3.2 对话类（2 张）

```sql
-- LLM 调用日志（所有 DeepSeek 调用都记录）
CREATE TABLE llm_log (
  id                INTEGER PRIMARY KEY,
  ts                TEXT NOT NULL,
  trigger           TEXT,              -- 'chat'/'fatigue_alert'/'debug'/'voice_chat'
  prompt            TEXT,
  response          TEXT,
  tokens_in         INTEGER DEFAULT 0,
  tokens_out        INTEGER DEFAULT 0,
  is_demo_seed      INTEGER DEFAULT 0,
  prompt_version_id INTEGER            -- FK to system_prompt_versions
);

-- 闲聊会话（只存 transcript + response，不存音频）
CREATE TABLE voice_sessions (
  id              INTEGER PRIMARY KEY,
  ts              TEXT NOT NULL,       -- ISO8601
  transcript      TEXT NOT NULL,       -- STT 转写结果
  response        TEXT,                -- LLM 回复
  summary         TEXT,                -- daemon 22:30 批量填充
  duration_s      REAL,
  trigger_src     TEXT,                -- 'chat'/'wake_word'/'fatigue_alert'
  is_demo_seed    INTEGER DEFAULT 0
);
```

### 3.3 智能演化类（2 张 + 1 张原配置）

```sql
-- 偏好演化史（追加写，每次偏好变化留一条）
CREATE TABLE preference_history (
  id              INTEGER PRIMARY KEY,
  ts              TEXT NOT NULL,
  field           TEXT NOT NULL,       -- 'fatigue_tolerance'/'target_muscle_groups'/'knee_caution'/...
  old_value       TEXT,
  new_value       TEXT NOT NULL,
  source          TEXT,                -- 'llm_inference'/'manual'/'seed'
  confidence      REAL,                -- 0.0-1.0
  is_demo_seed    INTEGER DEFAULT 0
);

-- 系统提示词版本（同时只有一个 active=1）
CREATE TABLE system_prompt_versions (
  id                     INTEGER PRIMARY KEY,
  ts                     TEXT NOT NULL,
  prompt_text            TEXT NOT NULL,
  based_on_summary_ids   TEXT,         -- JSON array: "[1,2,3]"
  active                 INTEGER DEFAULT 0,
  is_demo_seed           INTEGER DEFAULT 0
);

-- 用户偏好当前快照（key-value）
CREATE TABLE user_config (
  key         TEXT PRIMARY KEY,
  value       TEXT,
  updated_at  TEXT
);
-- 关键 keys:
--   fatigue_tolerance         medium / medium-low / low
--   target_muscle_groups      biceps,quadriceps
--   knee_caution              true / false
--   last_prompt_version       指向 system_prompt_versions.id
--   coach_style               鼓励 / 严厉
-- 兼容两套命名：key 'X' 和 'user_preference.X' 任一都行（读取时无前缀优先）
```

### 3.4 模型与可视化类（2 张，V4.9 新增）

```sql
-- 训练模型元数据（一行一个模型文件）
CREATE TABLE model_registry (
  id            INTEGER PRIMARY KEY AUTOINCREMENT,
  name          TEXT NOT NULL,       -- 'extreme_fusion_gru_curl'
  exercise      TEXT,                -- 'bicep_curl' / 'squat' / 'pose'
  path          TEXT NOT NULL,       -- 相对项目根的相对路径
  arch          TEXT,                -- 架构描述
  params_m      REAL,                -- 参数量（百万）
  size_kb       REAL,                -- 文件大小
  train_acc     REAL,                -- 0.0-1.0
  val_acc       REAL,
  epochs        INTEGER,
  dataset       TEXT,
  trained_at    TEXT,                -- ISO8601
  active        INTEGER DEFAULT 0,   -- 1=当前服役
  notes         TEXT,                -- 诚实标注（数据泄漏等）
  is_demo_seed  INTEGER DEFAULT 0
);

-- 预计算的 2D 投影给前端散点图
-- 一次训练把所有样本 7D→PCA/tSNE→(x,y)，按 label 着色展示可分性
CREATE TABLE feature_embeddings (
  id        INTEGER PRIMARY KEY AUTOINCREMENT,
  exercise  TEXT NOT NULL,
  label     TEXT NOT NULL,           -- 'standard'/'compensating'/'non_standard'
  x         REAL NOT NULL,
  y         REAL NOT NULL,
  source    TEXT,                    -- 'seed_pca'/'pca'/'tsne'
  notes     TEXT
);
```

## 5. 进程拓扑：谁读谁写

| 进程 | 启动方式 | 读 | 写 |
|---|---|---|---|
| `main_claw_loop.py` | `start_all_services.sh` | — | `training_sessions`, `rep_events` |

**重要**：WAL 模式下多写并发是安全的。但 `system_prompt_versions` 的 active 切换要原子 —— `create_system_prompt_version()` 内部用一个 transaction 做 `UPDATE active=0 THEN INSERT active=1`，不会出现双 active 或零 active 的中间态。

## 7. 数据库文件的同步规则（重要！）

`start_validation.sh` 的 rsync 命令里 **排除了 `data/` 目录**：
```bash
--exclude='data' \
```
这意味着 **开发机的 `data/ironbuddy.db` 不会同步到板端**，板端有自己独立的 `data/ironbuddy.db`。

**后果**：
- 在开发机跑 `seed_demo_2026_04.py` 只影响开发机
- 板端 DB 的 schema 可能缺少 voice_sessions / preference_history / system_prompt_versions 3 张新表
- 解决：需要在板端也跑一次 migration

**板端首次部署 schema 升级**（一次性）：
```bash
# 从 WSL 推 migration 到板端
scp -i ~/.ssh/id_rsa_toybrick \
  scripts/migrate_2026_04_20.sql \
  toybrick@10.18.76.224:/home/toybrick/streamer_v3/scripts/

# SSH 板端执行（板端的 sqlite3 必须可用）
ssh -i ~/.ssh/id_rsa_toybrick toybrick@10.18.76.224 \
  "cd /home/toybrick/streamer_v3 && \
   cp data/ironbuddy.db data/ironbuddy.db.bak_$(date +%s) && \
   sqlite3 data/ironbuddy.db < scripts/migrate_2026_04_20.sql"
```

演示种子不必同步到板端（板端是生产数据源，只用真实数据）。

## 10. OpenClaw Daemon 与偏好学习

### 10.1 为什么手动启
设计决策：**openclaw_daemon 必须手动启动**，不许并入 `start_all_services.sh`。
- 理由：daemon 依赖外网（DeepSeek API 或 OpenClaw Gateway WebSocket），断网会污染前端日志。
- 板端手动启：`ssh toybrick "bash /home/toybrick/streamer_v3/scripts/start_openclaw_daemon.sh"`
- 开发机：`bash scripts/run_openclaw_dev.sh`

### 10.2 触发文件协议
Daemon 主循环每 60s 扫 `/dev/shm/openclaw_trigger_*`：
```bash
touch /dev/shm/openclaw_trigger_daily_plan              # 立即生成今日计划 + 飞书早报
touch /dev/shm/openclaw_trigger_weekly_report           # 立即周报
touch /dev/shm/openclaw_trigger_preference_learning     # 立即偏好学习
```
消费完自动删除。**最多等 60 秒** 才见效。

### 10.3 规则引擎逻辑
`_compute_rule_based_preferences(db, today)` 当前规则：

| 字段 | 触发条件 | 值 | 置信度 |
|---|---|---|---|
| `knee_caution` | 当日 voice_sessions 包含"膝盖" ≥ 2 次 | `true` | 0.88 |
| `fatigue_tolerance` | 近 3 天 emg_target 平均下降 ≥ 20% | `medium-low` | 0.75 |
| `target_muscle_groups` | 当日训练 exercise 多样性 | 聚合字符串 | 0.60-0.72 |

规则代码在 [openclaw_daemon.py](/home/qq/projects/embedded-fullstack/hardware_engine/cognitive/openclaw_daemon.py)。未来要改规则直接改这里，改完 `bash scripts/stop_openclaw_dev.sh && bash scripts/run_openclaw_dev.sh` 重启生效。

### 10.4 动态 System Prompt 注入链
1. `openclaw_daemon` 规则引擎运行 → `_compose_prompt_text(snapshot)` 拼文本
2. 调 `db.create_system_prompt_version(prompt_text, summary_ids)` → 写新行 + active=1 + 旧 active=0 + `user_config.last_prompt_version` 更新
3. 下次 `voice_daemon._try_deepseek_chat()` 被调时，`get_active_system_prompt(fallback=...)` 返回最新文本
4. DeepSeek 以新 prompt 应答 → `llm_log` 记录 `prompt_version_id`

未来 debug 时，这些是最常用的 grep/read 入口：

```bash
# 看当前 active prompt 文本
python3 -c "import sys;sys.path.insert(0,'hardware_engine');from persistence.db import FitnessDB;d=FitnessDB();d.connect();print(d.get_active_system_prompt())"

# 看偏好快照
python3 -c "import sys;sys.path.insert(0,'hardware_engine');from persistence.db import FitnessDB;d=FitnessDB();d.connect();print(d.get_user_preferences_snapshot())"

# 看最近 5 次偏好变化
sqlite3 data/ironbuddy.db "SELECT ts,field,old_value,new_value,confidence FROM preference_history ORDER BY ts DESC LIMIT 5"

# 看最近 3 次 LLM 调用
sqlite3 data/ironbuddy.db "SELECT ts,trigger,substr(response,1,60) FROM llm_log ORDER BY ts DESC LIMIT 3"

# 看当前 active prompt id
sqlite3 data/ironbuddy.db "SELECT id,ts,substr(prompt_text,1,80) FROM system_prompt_versions WHERE active=1"

# 全部 8 张表行数
python3 scripts/demo_view.py overview
```
