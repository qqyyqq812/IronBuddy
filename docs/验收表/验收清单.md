> **说明**：配合刚才推送到板端的最新代码，供你在现场逐一验收打勾。

### 1️⃣ UI 界面纯净化验收
- [ ] **视觉免滑动**：手机访问主页，确认 5 大核心指标（包括缩进圆心的角度）全横排在一屏显示，且无滚动条。
- [ ] **静息波形熄灭**：不插传感器时不出现假波形扰动。
- [ ] **骨架画面修长**：骨架头部上沿不再越界被裁切；服务状态标签移至骨架下方。
- [ ] **残影清除**：点击“一键停止”后，确认残留的波形与骨架数据完全清空。
- [ ] **极简设置页**：确认旧版的“用户参数”模块已被彻底拔除。

### 2️⃣ 语音死锁与太监音验收
- [ ] **抢活死锁解除**：对它说“静音”，它播报好的；然后再说“解除静音”，确认不再陷入被自己封印的死循环。
- [ ] **硬警报闭嘴**：在静音模式下，故意做一个违规动作，确认它绝对不强行播报警报。
- [ ] **电音杂音消除**：让它说长句子，确认 `arecord` 和 `play_audio` 的争抢 Bug 已消失，声音清晰不断片。
- [ ] **断句容忍度**：大喊“教练”后稍微顿挫，确认单次问答逻辑足够宽容。

### 3️⃣ LLM 与疲劳触发验收
- [ ] **红线动态触点**：通过 API/UI 将危险红线设为 `1900`。
- [ ] **动态数值播报**：发力超越 1900 时，确认 DeepSeek 触发且教练语音确实报出了 `1900` 这个数值，而不是旧版的 `1500`。
- [ ] **疲劳原子清空**：云端点评结束后，疲劳积分立即归零。

## §5 风险矩阵与降级阶梯

| 风险 | 概率 | 影响 | 应对 |
|------|------|------|------|
| 现场 WiFi 卡 → Cloud RTMPose 断线 | 中 | 视觉 FPS 掉 | UI 切 Local NPU（`/dev/shm/vision_mode.json`=`local`）；画面略慢但可用 |
| MIA tar 扫描超时（52.9 GB 扫 20 min 内不完成） | 中 | MIA 方案回退至无外部数据 | 分批扫（按 Subject 范围）或手动 tar 抽取 |
| MIA 深蹲 label 不均衡（GoodSquat 远少于 BadSquat 或反之） | 中 | 训练偏向一类 | class_weight balanced + oversample |
| switch_model.sh 切权重后未重启 FSM 进程 | 低 | 旧模型残留 | 脚本内 pkill + nohup 重启 |
| 周一重训 loss 不收敛 | 中 | GRU 分类浮窗乱跳 | 用历史权重兜底：`cp models/extreme_fusion_gru.pt.bak hardware_engine/extreme_fusion_gru.pt`（周日前做一份备份） |
| 摄像头 pose_score 持续 < 0.10 | 低 | 采集 CSV 过少 | 调整距离 1.5-2m + 补光灯 + 动作幅度放大 |
| EMG UDP 丢包 | 低 | 波形平线 | 重启 `udp_emg_server.py`；`bash scripts/stop_collect.sh && start_collect.sh` |
| DeepSeek API 限流 | 极低 | 自动教练无回复 | 视频里用按钮触发一次就够，不演示 3 路触发 |
