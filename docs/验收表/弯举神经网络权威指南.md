## 1. 30 秒速读

**系统定位**：边缘板端 AI 健身教练的弯举评判子系统。输入 = (摄像头骨架角度 + 双通道 EMG)，输出 = (three-class classification + similarity)，驱动 UI 计数 / 骨架染色 / 语音播报。

**三大状态**：
- **pure_vision**：仅视觉 FSM 角度硬判定（`peak_angle < 50°` → good, 否则 failed）。GRU 不推理。
- **vision_sensor**：FSM 只负责 rep 边界检测 + 疲劳累加。**good / failed / comp 完全由 GRU 三分类决定**。
- 通过语音"纯视觉" / "视觉加传感"切换，写 `/dev/shm/inference_mode.json`。

**模型**：`CompensationGRU` (7D × 30 帧窗口，hidden=16，GRU+cls+sim+phase 四头，**11 KB**)。本地真录（3 条）+ 增强副本（33 条）训练，三分类：standard (0) / compensating (1) / non_standard (2)。

**标签映射**（与深蹲唯一不同点，因弯举 "bad" 语义是摆臂借力）：
- `golden` → **standard** (CLASS_GOLDEN=0)
- `bad` → **compensating** (CLASS_LAZY=1)  ← 弯举 bad = 摆臂借力
- `lazy` → **non_standard** (CLASS_BAD=2)  ← 弯举 lazy = 收缩幅度不足

**训练产出**：`hardware_engine/extreme_fusion_gru_bicep.pt`（与深蹲独立）。

**当前状态（V1.0）**：train acc 99.8%，混淆矩阵清洁，三类自检（从真实 CSV 抽样）全绿。

### 并行 EMG 数据流

```
┌───────────────────────────────────────────────────────────────────────┐
│ 真实 ESP32         OR        simulate_emg_from_bicep.py                │
│   (未接)                        读实时角度 → 三 label 波形锚点插值    │
│                                 → ±15% 噪声 → 反 domain_calibration    │
│                                 → 合成 UDP ASCII "target comp\n"       │
│                                 → UDP 127.0.0.1:8080                   │
└───────────────────────────────────────────────────────────────────────┘
                                   │
                                   ▼
              udp_emg_server (port 8080)
                DSP: HP20 + Notch50 + LP150
                → 100-sample rolling RMS
                → / MVC_BASE × 100 → 0-100 pct
                → domain_calib 正变换 α*pct + β → clip [0,100]
                → io_dumper @ 50Hz 写 /dev/shm/muscle_activation.json
                → **exercise=bicep_curl 分支**:
                     biceps ← target_pct
                     glutes ← comp_pct   ← 弯举下代偿肌 slot
                                   │
                                   ▼
           main_claw_loop.py 读 muscle_activation.biceps + glutes
              → _gru_feature_buf.append([..., target=biceps, comp=glutes, ...])
              → rep 完成时 GRU.infer() → extreme_fusion_gru_bicep.pt
```

## 5. 数据集（本地真录 + 增强）

### 5.1 来源

用户自己录制的弯举视频，通过 `tools/collect_training_data.py --exercise bicep_curl --mode <label>` 在板端实时采集生成，每条 CSV 对应一次 ~50s 连续动作（约 25 rep）。

### 5.2 本地目录

```
data/bicep_curl/
├── golden/  1 条 CSV  (2026-04-12 091807, ~956 行)
├── bad/     1 条 CSV  (2026-04-12 092152, ~1047 行)
└── lazy/    1 条 CSV  (2026-04-12 092347, ~1050 行)

data/bicep_curl_augmented/         (tools/augment_curl_data.py 产出)
├── golden/  11 条 CSV  (1 seed + 10 aug, 共 ~10516 行)
├── bad/     11 条 CSV  (共 ~11517 行)
└── lazy/    11 条 CSV  (共 ~11550 行)
```

合计：**36 条 CSV / ~33000 行 / 3564 个 30 帧滑窗样本**。

### 5.3 CSV 10 列 Schema

```
Timestamp, Ang_Vel, Angle, Ang_Accel, Target_RMS, Comp_RMS, Symmetry_Score, Phase_Progress, pose_score, label
```

训练使用前 8 列（忽略 pose_score），label 列用于目录映射交叉校验。

| 列 | 单位 / 范围 | 弯举含义 |
|----|-------------|----------|
| Angle | 度 [10°, 175°] | 肩-肘-腕三点法肘关节角 |
| Ang_Vel | 度/frame | `angle[t] - angle[t-1]`（来自 FSM `_angle_history` 差分）|
| Ang_Accel | 度/frame² | `ang_vel[t] - ang_vel[t-1]` |
| Target_RMS | 0–100 pct | **biceps** (肱二头肌) 经 udp_emg 归一化 |
| Comp_RMS | 0–100 pct | **forearm/三角肌** 代偿肌 slot（udp_emg 写到 glutes 键） |
| Symmetry_Score | 0–1 | 左右臂 RMS 对称度 |
| Phase_Progress | 0–1 | FSM 相位（离心/向心）|

### 5.4 数据增强规则（`tools/augment_curl_data.py`）

- 时间 warp：1D 线性插值，因子 ±10%
- 幅度：Target/Comp 乘法 ×±15%
- 噪声：Target/Comp 加 N(0, 1.5) clip [0, 100]
- 几何列（Angle/Ang_Vel/Ang_Accel/Symmetry/Phase）**保留不变**
- 默认 `--factor 10` → 11× 扩增（1 seed + 10 aug per CSV）

### 5.5 标签语义重映射（**与深蹲的关键差异**）

| 用户标签（录制时 CLI `--mode`）| CSV label 列 | 模型类别 | 物理含义 |
|--------------------------------|--------------|----------|----------|
| `--mode golden` | "golden" | **CLASS_GOLDEN (0) = standard** | 肩稳定、肘固定、肱二头主导收缩到 <50° |
| `--mode bad`    | "bad"    | **CLASS_LAZY (1) = compensating** | 摆臂借力 / 耸肩 / 躯干后仰 → comp 肌（前臂/三角肌）峰值 |
| `--mode lazy`   | "lazy"   | **CLASS_BAD (2) = non_standard** | 收缩幅度不足（顶峰角度 >60°），target/comp 双低 |

对照深蹲侧（`golden→0, lazy→1 compensating(合成), bad→2 non_standard`）：弯举的 bad 与 lazy 语义**对调**。

### 6.3 训练细节

- **模型**：`CompensationGRU(input_size=7, hidden_size=16, num_layers=1)` — 与深蹲共用同一 `fusion_model.py` 定义
- **损失**：`0.3 × MSE(similarity) + 0.7 × CE(3-class)`
- **Similarity 目标**：standard→1.0, compensating→0.6, non_standard→0.3
- **类别权重**：反比于样本数（`weights[i] = total / (3 × count[i])`）
- **Scheduler**：`StepLR(step=8, gamma=0.5)`
- **滑窗**：30 帧 × stride 10，36 条 CSV → 3564 样本
- **耗时**：CPU 约 20 秒（WSL i7）

### 6.4 归一化（**训练/推理必须完全一致**）

```python
def _normalize_inplace(features):

    features[:, 4] = clip(features[:, 4], 0.0, 100.0) / 100.0  # Comp_RMS (clip 关键)
    # Symmetry / Phase 已是 0-1
```

## 7. 推理管线 & 归一化契约

### 7.1 模型架构

复用深蹲侧的 `CompensationGRU(input=7, hidden=16, layers=1)`，输出 4 头：
- similarity ∈ [0, 1]
- classification ∈ {"standard", "compensating", "non_standard"}
- confidence ∈ [0, 1]
- phase ∈ {"standing", "descending", "bottom", "ascending"}（对弯举语义映射：伸直 / 收缩 / 顶峰 / 下放）

### 7.2 推理入口（`main_claw_loop.py:906-946`）

GRU 推理仅在：
1. `inference_mode == "vision_sensor"`
2. `_cur_reps > fsm._prev_total_reps_gru`（rep 边界推进）
3. `_GRU_MODEL is not None`（加载成功）
4. `len(_gru_feature_buf) >= 30`

### 7.3 推理端特征构造（`main_claw_loop.py:805-864`）

```python
# EMG key 按 exercise 路由（V1.0 补丁 D）
if current_exercise == "bicep_curl":
    target_emg = acts.get("biceps", 0.0)    # udp_emg: biceps ← target_pct
    comp_emg   = acts.get("glutes", 0.0)    # udp_emg: glutes ← comp_pct
else:
    target_emg = acts.get("glutes", 0.0)    # 深蹲原语义
    comp_emg   = acts.get("biceps", 0.0)

# 7D 特征 append
_gru_feature_buf.append([
    ang_vel, angle, ang_accel,
    target_emg, comp_emg,
    1.0,            # Symmetry 占位（暂未算左右对称）
    phase_prog,     # 从 _angle_history min/max 估计
])

# rep 完成时推理
window = np.array(_gru_feature_buf[-30:], dtype=np.float32)
window[:, 1] /= 180.0
window[:, 3] /= 100.0
window[:, 4] /= 100.0
window[:, 2]  = np.clip(window[:, 2] / 10.0, -1.0, 1.0)
# Ang_Vel (col 0) 推理端 /30 归一化 — V7.15 对齐

```

## 8. FSM 解耦（pure_vision vs vision_sensor）

### 8.1 `DumbbellCurlFSM` 三计数器（V7.15）

| 字段 | 含义 | 推进时机 |
|-----|------|---------|
| `_good_reps` | standard rep 数 | **pure_vision**: peak<50°; **vision_sensor**: GRU=standard |
| `_failed_reps` | non_standard rep 数 | **pure_vision**: peak>50°; **vision_sensor**: GRU=non_standard |
| `_compensation_count` | compensating rep 数 | 仅 vision_sensor: GRU=compensating（带 `_compensation_last_rep` 防重复）|
| `_total_reps_count` | **总 rep 数**（边界，模式无关）| CURLING→EXTENDING 过渡时 += 1 |
| `_prev_total_reps_gru` | GRU 推理路径 rep 进度 | 推理触发后同步 |
| `total_fatigue_volume` | 疲劳积分 | **所有** rep 完成时 +1500/7 ≈ 214.3 |

### 8.2 关键参数

- `ANGLE_STANDARD = 50°`（顶峰收缩合格阈值）
- 进入 CURLING：`angle < 140°`
- 结算 EXTENDING：`angle > 145°`
- 关键点最低置信度：`MIN_KPT_CONF = 0.05`（NPU uint8 模型）

### 8.3 顶峰角度外插补偿（V7.13 + V7.14）

CURLING 分支在每帧算 min 角度之前，若上帧在快速收紧（`last_ang_vel_cu < -8.0`）且两帧间隙 80-250ms，估算漏帧顶峰：

```python
predicted = last_angle + last_ang_vel * (dt * 0.5)
virtual_peak = max(25.0, predicted)   # 肘关节最小闭合角 ~25°
self._min_angle_in_rep = min(min_so_far, virtual_peak, angle)
```

### 8.4 模式切换运行时

```python
# FSM.update() CURLING→EXTENDING rep 结算里读 inference_mode (100ms 缓存)
if self._mode_cache == "vision_sensor":
    logging.info(f"⏳ [vision_sensor] 弯举 rep #{self._total_reps_count} 顶峰{bottom:.0f}° 等待 GRU 分类")
else:
    # pure_vision: 保留角度硬判定
    if bottom < ANGLE_STANDARD: self._good_reps += 1
    else:                       self._failed_reps += 1; trigger_buzzer_alert()
```

## 11. 历史根因 & 版本

### V1.0（2026-04-20）本文件初版。解决的问题：

1. **弯举无专用三分类训练脚本**

   - 症状：从深蹲切到弯举后，第一个弯举 rep 用的是深蹲残留的 7D 特征滑窗
   - 修复：`_gru_feature_buf.clear()` 放在 FSM 切换时

本弯举权威指南所有 FSM 解耦 / 模式闸门 / EMG scope / updateRig comp 参数等功能均依赖 V7.13-V7.15 的深蹲侧根因修复（见 `深蹲神经网络权威指南.md` §11）。弯举代码无需再做这些通用基建修复，**只做弯举专属差异**。

## 13. 常见 Debug 场景

### 13.1 切弯举后 UI 一直"无模型"

**症状**：`state_feed` 无 `classification` 字段，UI 显示"无模型"。  
**诊断**：
```bash
tail -30 /tmp/fsm_loop.log | grep -E "GRU|exercise|Loaded"
```
**可能原因**：
- `extreme_fusion_gru_bicep.pt` 未上板 → 补 rsync
- `_load_gru_model("bicep_curl")` 回退到 squat 权重（查 log "Loaded .../extreme_fusion_gru.pt"）→ 权重文件名或路径错
- `_gru_feature_buf` 未清空残留（切换后第一个 rep 用的是深蹲特征）→ 看 log 有无 "动作模式切换"

### 13.2 GRU 一直输出同一类

**诊断**：
```bash
python3 tools/train_gru_three_class_bicep.py --selftest-only
```
self-test 失败 → 权重坏，重训。  
self-test 成功但板端不对 → 归一化不一致或 EMG key 路由错。查 `/dev/shm/muscle_activation.json` 是否有 `biceps`/`glutes` 非零字段。

### 13.3 代偿骨架不黄闪

- `state_feed.comp` 是否递增：`curl -s http://10.18.76.224:5000/state_feed | python3 -m json.tool | grep comp`
- `updateRig` 第 7 参 comp 是否传入（`index.html` updateRig 调用处）
- `_lastClassification === 'compensating'` 是否更新

### 13.4 EMG 曲线直线

见深蹲权威指南 §13.4。与弯举无差异。

### 13.5 弯举分类偶尔误判为深蹲标签

**症状**：语音切弯举后，第一个 rep 分类结果异常。  
**原因**：`_gru_feature_buf` 未清空（V1.0 已修），或 udp_emg_server 的 `user_profile.json` 未切换。  
**诊断**：
```bash
cat /dev/shm/user_profile.json   # 应是 {"exercise":"bicep_curl"}
```
