# IronBuddy V3.0 巅峰架构：双轨智算系统与核心防雷白皮书

> [!IMPORTANT]
> **归档说明**: 本文档剥离了早期的开发妥协与次优解。以下纲领确立了基于 `RK3399ProX` 本地生态算力极限的**最核心、最成熟的软硬件架构（State-of-the-Art）**。此白皮书作为参赛定海神针及后续**视频拍摄录制**唯一尊奉的标准。

---

## 模块一：NPU 视觉中枢与算力极化方案 (`yolov_engine` -> `RTMPose`)

> [!TIP]
> **最新战略：废弃 V5/V8 妥协，拥抱边缘算力之王 RTMPose**

早期尝试了对硬件不友好的 3D lifting 算法和具有“自体穿透遮挡抖动（Occlusion Jitter）”的 YOLO 模型，引发了误判灾难。
*   **定锤算力**：RK3399ProX 携带 3.0 TOPS 算力。我们将模型彻底更换为 **OpenMMLab 出品的 RTMPose-m（基于 RKNN INT8量化）**。
*   **技术红利**：RTMPose 是当今为低功耗 ARM NPU 专属定制的骨架追踪之王。它能彻底解决运动者**蹲底时大腿遮挡导致的抖动乱数**，不仅帧率可以硬生生拉到 30 FPS 的上限，其**骨架点坐标极高的拓扑刚性**将为我们的状态机提供绝对无杂音的“净水信号”。

---

## 模块二：表现层与零摩擦 UI 渲染 (`streamer_app` & `2.5D CSS Rig`)

> [!TIP]
> **最新战略：摒弃重度的 2D-to-3D 深度解算网络，采用原生 CSS3D 矩阵零损耗渲染侧向视角**
> *(回应批注：旧版 2D转3D 是用 Python 跑一个神经网络强制推算 Z 轴深度，吃算力且有算法延迟。现在的 2.5D CSS 则是属于“前端视觉魔术”：直接把骨骼角度传给浏览器的 GPU，做 0ms 原生拉伸旋转。一个是后端重度数学推理，一个是纯粹的前端0延迟极速映射。)*

系统交互不追求花哨堆栈，而是追求极致低延迟跟手。
*   **交互逻辑**：前端不再读取耗时的三维估算网格，而是通过轮询 `RK3399 RAMDisk (/dev/shm)`。
*   **物理动态还原 (深蹲限定)**：重置原本像“一张被上下拍扁的纸片”的旧版 SVG 人偶。我们将提取 RTMPose 算出的肩骨盆前倾斜率，注入 **完全 0ms 算力开销的 CSS DOM 矩阵 (`transform: translateY() rotateZ()`)**。
> *(回应批注：这**不是**可以跟你做任意舞蹈动作的全局重定向模型！它是一个**专为骨盆、躯干和膝盖硬编码的“深蹲语义映射器”**。你蹲多低、上身倾斜多少度，小人都会**毫无延迟的连续跟进**，获得极度跟手的真实物理拉扯感，但仅限深蹲与类似的发力姿态。)*

> [!WARNING]
> **环境保障 (视频流控)**：由于我们的核心交付物是**视频拍摄展示**（非混乱的现场展台），我们无需过度担心局域网公用频段踩踏导致 MJPEG 抢网。但在单机录制机位上，必须确保 RK3399 宿主路由器的信号绝对稳定，以保障高质量素材收录。

## 模块三：状态机与系统灵魂 —— 视觉+肌电双模融合评判（战略高地）

> [!TIP]
> **最新战略：摒弃“死算次数”，建立以“疲劳与代偿”为主轴的医学级判据**

这是整套 IronBuddy 系统的**终极壁垒**。我们不再使用低级的 If-Else 角度计数，而是采用 **宏观动作捕捉（CV）+ 微观发力特征（外挂 EMG 肌电模块）** 串联的高级判定环路：

1. **第一层：动作有效性与代偿（借力）抓捕**
   * **[合格帧]**：RTMPose 判断大腿平行（膝盖折叠 < 90 度）时，传感器回传的目标肌群（股四头/臀肌）由于受迫发力包络激增。此为好球，屏幕小人的肌群将**荡起激烈的绿色充血波纹**！
   * **[动作走形（作弊判定）]**：虽然你蹲下去起来了，但 EMG 传感器捕获到你的非发力肌群（如竖脊肌、腰方肌）在动作周期内爆发了尖峰脉冲。这意味着你**核心崩塌，利用下背部借力代偿**。此时：状态机直接否决本次有效性，屏幕爆出**震慑性红光**，并触发板载蜂鸣器发声警报。
2. **第二层：终结机制重塑（真实运动阈值）**
   * **不再是做满 10 个算一组**。系统会不停积分单次合格深蹲中的“肌电总包络积”。
   * 小人表皮会随着训练量慢慢变成**恐怖的深红色（疲劳累积）**。
   * 当**实际做功总阈值（Total Fatigue Volume）达到死线**（如积攒 1500 强度），系统立即强行切断组次，宣布你极限完成，并将此组波峰谷值压缩发送给 AI！

## 模块四 & 模块五：外部触角与皮层预警 (Voice & OpenClaw)

> *(此两项保持极客架构不变，确保抗压定力)*

*   **大模型通信 (`openclaw_bridge.py`)**：已用完全异构的 UUID Session 网关（TCP 18789）打通 DeepSeek。利用**异步多进程文件监听机制**，就算大模型死机、卡死 60 秒拿不到回复，RK3399 端侧的 3D推理和前端运动 UI 也绝对不会卡顿一帧！
*   **语音监听 (`voice_daemon.py`)**：ALSA + Vosk 本地唤醒是绝对零网络依赖的。

> [!WARNING]
> **录制拍摄环境配置：** 既然我们是以**拍摄视频**为最终展示形态，那么云端 Edge-TTS 被完全解放！我们不仅不用再保守地掐断云端合成器（不用担心展会物理断网），反而可以在录制前把 `voice_daemon.py` 的合成音色直接换成最激昂、最高清的云端教练音色，力求视频视听效果的极致震撼与渲染力。
