## §1 V4.2 训练架构总览

### §1.1 八条锁定决策

1. **融合粒度**：rep-level（每 rep 完成一次推理），不是 frame-level，不是 logit-level 晚融合。
2. **融合结构**：两 Encoder 各出 8d embedding + 5 个手工标量 → `Linear(21, 3) = 66 参数` 分类器。紧凑中融合。
3. **参数预算红线**：LOSO 每折 90 样本训练，融合头参数 ≤ 200（欠拟合救援才解锁 203），训练集 F1 − LOSO 验证 F1 差 ≤ 0.15。
4. **Encoder 预训练**：本地 270 rep 自监督 masked AE + 3 倍数据增强。**拒绝伪迁移**。
5. **Ninapro / Camargo 用途**：L3 only（噪声谱数据增强 + J1/J2 统计基准），不进训练循环。深蹲 Ninapro 彻底弃用。
6. **双动作贴片**：弯举 CH0 肱二头肌 + CH1 前臂屈肌群；深蹲 CH0 股直肌 + CH1 腰部竖脊肌。
7. **三类标签**：0 standard / 1 compensation（甩臂或腰背借力，融合头核心靶点）/ 2 bad_form（幅度不到位）。
8. **异常检测**：embedding cosine 相似度 + L2 距离，超训练集 95 百分位 → 橙色 banner + 语音「请检查电极」，独立于分类路径。

### §1.2 架构示意

```
         ┌────────────────────┐         ┌────────────────────┐
视觉序列 │  VisionEncoder     │         │   EMGEncoder       │ EMG 序列
30×4d →  │  (4, 16, 8)        │         │   (7, 16, 8)       │ ← 200×7d
         │  ~200 参数         │         │   ~400 参数        │
         └──────┬─────────────┘         └───────┬────────────┘
                │                                │
                │  vision_emb (8d)         emg_emb (8d)
                │                                │
                └──────────┬─────────────────────┘
                           │
                  ┌────────▼──────────┐       ┌─────────────────────┐
                  │ 21d 拼接           │ ←──── │ HandCraftedFeatures │
                  │ vision(8)+emg(8)   │       │ 5 手工标量(0 参数)  │
                  │  + 标量(5)         │       └─────────────────────┘
                  └────────┬──────────┘
                           │
                  ┌────────▼──────────┐
                  │ FusionHead        │
                  │ Linear(21, 3)     │
                  │ = 66 参数         │
                  └────────┬──────────┘
                           │
                  softmax → {0 standard, 1 compensation, 2 bad_form}
```

整模型参数合计约 690，远低于预算上限 800。实现见 [`hardware_engine/cognitive/dual_branch_fusion.py`](../../hardware_engine/cognitive/dual_branch_fusion.py)。

### §1.3 为什么是 21 维

| 维度段 | 大小 | 来源 | 物理意义 |
|---|---|---|---|
| vision_emb | 8 | `VisionEncoder(30×4)` masked AE | 动作轨迹紧致表示 |
| emg_emb | 8 | `EMGEncoder(200×7)` masked AE | 肌电时频紧致表示 |
| emg_target_comp_ratio | 1 | 手工：`target.max() / (comp.max()+ε)` | 代偿强度 |
| cos_sim(v_emb, e_emb) | 1 | 手工：两 Encoder 一致性 | 模态对齐（异常检测也复用） |
| phase_progress_max | 1 | 手工：FSM 相位进度峰值 | 幅度到位度（bad_form 信号） |
| angular_acc_peak_abs | 1 | 手工：rep 前 30% 角加速度峰值 | 弯举甩臂线索 |
| torso_forward_tilt_peak | 1 | 手工：躯干前倾峰值 | 深蹲弯腰线索 |

**为什么手工标量不让 Encoder 自己学**：LOSO 每折只有 90 样本。手工标量是 biomechanics prior（先验），直接给模型省下学习成本，避免小样本上 Encoder 瞎拟合。

## §2 MVC 校准（SENIAM 2000 协议）

权威引用：Hermens H.J. et al. (2000)，SENIAM 项目（www.seniam.org，Google Scholar 引用 > 6000）。

### §2.1 两块肌肉操作细则

| 项 | 肱二头肌（弯举 CH0） | 股直肌（深蹲 CH0） |
|---|---|---|
| 体位 | 坐姿，上臂贴躯干 | 坐姿，大腿水平 |
| 关节角 | 肘 90°，前臂旋后 | 膝 60° |
| 阻力 | 抗最大等长屈肘 | 抗最大等长伸膝 |
| 持续 | 3–5 秒最大发力 | 3–5 秒最大发力 |
| 次数 | 3 次，取最大峰值 | 3 次，取最大峰值 |
| 间隔 | ≥ 2 分钟 | ≥ 2 分钟 |
| 电极间距 | 20 mm（SENIAM 标准） | 20 mm |
| 皮肤阻抗 | 酒精擦拭 + 轻磨 < 5 kΩ | 同 |

### §2.2 三红线（违反即重测）

1. **皮肤必处理**：酒精擦拭 + 轻磨至红润。阻抗 < 5 kΩ 方可贴片。
2. **取最大不取平均**：3 次 MVC 取峰值，不取均值。平均会低估最大随意收缩能力。
3. **标准差 > 20% 均值则重测**：3 次之间差异过大说明某次没全力。

### §2.3 输出 JSON Schema

写入 `/dev/shm/emg_calibration.json` 以及 `data/v42/user_XX/mvc_calibration.json`：

```json
{
  "protocol": "SENIAM-2000",
  "exercise": "curl",
  "user_id": "user_01",
  "peak_mvc": {
    "ch0": 847.3,
    "ch1": 612.1
  },
  "peak_mvc_trials": {
    "ch0": [820.5, 847.3, 832.1],
    "ch1": [598.2, 612.1, 604.7]
  },
  "std_pct": {"ch0": 0.08, "ch1": 0.11},
  "electrode_impedance_kohm": {"ch0": 3.2, "ch1": 4.1},
  "ts": "2026-04-18T09:45:00+08:00"
}
```

> `std_pct` 是 3 次峰值的 std/mean。任何通道 `std_pct > 0.20` 必须整条重测。

## §3 11D 时序特征定义

每 rep 采样到的 CSV 共 **13 列** = 11 特征 + Timestamp + label。特征维度与归一化规则如下。

| # | 特征名 | 单位 | 归一化 | 来源通道 | 物理意义 |
|---|---|---|---|---|---|
| 1 | `Ang_Vel` | rad/s | 无 | 视觉骨架 | 关节角速度（弯举肘 / 深蹲膝） |
| 2 | `Angle` | rad | 无 | 视觉骨架 | 关节角度 |
| 3 | `Ang_Accel` | rad/s² | 无 | 视觉骨架 | 角加速度（甩臂信号） |
| 4 | `Target_RMS_Norm` | — | ÷ `peak_mvc.ch0` | EMG CH0 | 目标肌（肱二头 / 股直肌）RMS |
| 5 | `Comp_RMS_Norm` | — | ÷ `peak_mvc.ch1` | EMG CH1 | 代偿肌（前臂 / 竖脊肌）RMS |
| 6 | `Symmetry_Score` | — | 0–1 | 视觉左右骨架 | 左右对称比 |
| 7 | `Phase_Progress` | — | 0–1 | FSM | 相位进度（rep 内位置） |
| 8 | `Target_MDF` | Hz | ÷ 100 | EMG CH0 | 中位频率（疲劳指标） |
| 9 | `Target_MNF` | Hz | ÷ 150 | EMG CH0 | 平均频率 |
| 10 | `Target_ZCR` | — | ÷ 400 | EMG CH0 | 过零率（高频噪声信号） |
| 11 | `Target_Raw_Unfilt` | ADC | ÷ 2048 | EMG CH0 原始 | 未滤波 RMS（12-bit ADC 饱和检测） |

### §3.1 归一化公式

- `Target_RMS_Norm = RMS_ch0(window) / peak_mvc.ch0`（window 默认 5 ms 滑窗）
- `Target_MDF_Norm = MDF(window) / 100.0`（MDF 计算见 [`docs/technical/sEMG泛化实现指南.md §3`](sEMG泛化实现指南.md)）
- `Target_Raw_Unfilt_Norm = RMS_raw / 2048`（2048 = 12-bit ADC 中值，不饱和时 ≤ 1）

### §3.2 数据契约

CSV 完整表头（写入时务必保序）：
```
Timestamp, Ang_Vel, Angle, Ang_Accel,
Target_RMS_Norm, Comp_RMS_Norm, Symmetry_Score, Phase_Progress,
Target_MDF, Target_MNF, Target_ZCR, Target_Raw_Unfilt,
label
```

- 单 rep CSV 约 200 行（EMG 200Hz × 1 秒典型 rep 时长）。视觉 30fps 在 EMG 帧上做最近邻对齐。
- `label` 列在采集时由 `--label {standard,compensation,bad_form}` 固定填入 0/1/2。
- 无 NaN / Inf，否则由 [`tools/validate_v42_dataset.py`](../../tools/validate_v42_dataset.py) 拒收。

## §4 三类标签语义

### §4.1 哑铃弯举

- CH0 肱二头肌（发力点）：肌腹中点，肘窝到肩峰 1/3 处
- CH1 桡侧腕屈肌 / 前臂屈肌群（代偿点）：肘窝下 5 cm 前臂掌侧

| 标签 | 动作语义 | 视觉特征 | EMG 特征 | 推理路径 |
|---|---|---|---|---|
| 0 standard | 肘角 170°→45° 流畅，速度稳 | 角速度适中 | 肱二头肌强、前臂低 | 单模态（视觉） |
| **1 compensation** | **甩臂借惯性** | **角加速度尖峰**、峰速偏高 | **前臂 RMS 尖峰** + 肱二头肌相对弱 | **跨模态⭐** |
| 2 bad_form | 幅度不到位（部分 rep） | `Phase_Progress_max < 0.85` | 两通道都温和偏低 | 单模态（视觉） |

### §4.2 深蹲

- CH0 股直肌（发力点）：髂前上棘到髌骨上缘连线 1/2 处
- CH1 腰部竖脊肌（代偿点）：L1-L3 水平两侧 3 cm 旁开

| 标签 | 动作语义 | 视觉特征 | EMG 特征 | 推理路径 |
|---|---|---|---|---|
| 0 standard | 膝角到 ~90°，躯干直 | 正常 | 股直肌强，竖脊肌仅底部温和 | 单模态 |
| **1 compensation** | **腰背借力站起** | 躯干前倾 > 45° | **竖脊肌 RMS 尖峰**（上升段）+ 股直肌弱 | **跨模态⭐** |
| 2 bad_form | 深度不够（半深蹲） | 膝角未到 100° / `Phase_Progress_max < 0.85` | 两通道都温和偏低 | 单模态（视觉） |

### §4.3 为什么 label 1 compensation 必须跨模态

视觉维度上看，**甩臂弯举的末端位置和标准弯举一模一样**——肘角都到了 45°，关键帧的 pose 无差别。单凭视觉只能通过角加速度尖峰间接推断，且容易被用户节奏变化混淆。

EMG 维度才暴露事实：甩臂时前臂屈肌群为了抓紧哑铃先发力，在 rep 前 30% 出现 RMS 尖峰；同时肱二头肌因为借了惯性，整条 rep 的积分明显弱于 standard。

**单模态视觉模型在 label 1 上召回天花板约 50-60%；融合头靠 `emg_target_comp_ratio` + `angular_acc_peak_abs` 两标量把天花板推到 70% 以上**。这是整套架构的立足点。

### §4.4 标注策略

**采集时人工指定 label，不做事后自动标注**。原因：
- 事后自动标注需要「金标准」规则，而规则如果能写出来，融合头就没必要训了。
- 让被采集人**刻意做代偿/不到位动作**，比对标准动作后事后判别更稳。
- 每类 15 rep × 3 组（三类），总共 45 rep × 2 动作 = 90 rep/人。

## §5 数据采集 SOP

### §5.1 步骤 1：贴片

按 §4.1 / §4.2 的电极位置，酒精棉擦拭皮肤两次，轻磨至皮肤微红。SENIAM 标准电极间距 20 mm，肌腹方向沿肌纤维走向。

### §5.2 步骤 2：阻抗测试

皮肤处理后使用 ESP32 固件阻抗自检功能或万用表测 CH0/CH1 对地阻抗，必须 < 5 kΩ。超标则继续擦磨，严重则换电极。

### §5.3 步骤 3：MVC 校准

浏览器打开板端 `http://板ip:5000`，点击「开始 MVC 校准」模态框，按提示完成 3 次 × 3–5 秒最大发力 × 间隔 ≥ 2 分钟。系统写入 `/dev/shm/emg_calibration.json`（schema 见 §2.3）。

若 `std_pct > 0.20` 或任一阻抗 > 5 kΩ，模态框会拒绝通过。

### §5.4 步骤 4：三类 × 15 rep × 2 动作

每人每动作每类 15 rep，两动作三类，合计 **90 rep/人 × 3 人 = 270 rep**。

```bash
# 弯举三类（每类 15 rep，执行时间约 20 分钟含指导）
python tools/collect_training_data_v42.py --user user_01 --exercise curl --label standard --reps 15
python tools/collect_training_data_v42.py --user user_01 --exercise curl --label compensation --reps 15
python tools/collect_training_data_v42.py --user user_01 --exercise curl --label bad_form --reps 15

# 深蹲三类
python tools/collect_training_data_v42.py --user user_01 --exercise squat --label standard --reps 15
python tools/collect_training_data_v42.py --user user_01 --exercise squat --label compensation --reps 15
python tools/collect_training_data_v42.py --user user_01 --exercise squat --label bad_form --reps 15

# 验证采集完整性
python tools/validate_v42_dataset.py --data-root data/v42
```

产出目录结构：
```
data/v42/user_01/
├── anthropometry.json       # 身高/上臂围/大腿围
├── mvc_calibration.json     # §2.3 schema
├── curl/
│   ├── standard/rep_001.csv ... rep_015.csv
│   ├── compensation/rep_001.csv ... rep_015.csv
│   └── bad_form/rep_001.csv ... rep_015.csv
└── squat/（同结构）
```

### §5.5 会话后处理

- 一人一 session 后用生理盐水擦电极控湿度。
- 每人结束立即跑 `validate_v42_dataset.py` 快照，发现任何 fail 立即补采集。
- 旧 `data/{bicep_curl,squat}/*` 保留但**不合并**（标签体系不同 + MVC 缺失）。

## §6 训练三阶段

### §6.1 Stage 1 · Encoder 自监督预训练

```bash
python tools/pretrain_encoders.py --exercise both --epochs 30 --data-root data/v42
```

- 输入：本地 270 rep（弯举 135 + 深蹲 135）
- 数据增强 3 倍：时间扭曲 ±10% + 幅度抖动 ±5% + Ninapro 高频噪声叠加（L3 用途，缺失则自动跳过）
- 任务：random mask 30% 时间点，MSE 重建
- 超参：Adam lr=1e-3, batch=32, epoch=30
- 期望：CPU 约 10 分钟，loss 从 ~0.5 下降到 ~0.1
- 产出：
  - `hardware_engine/cognitive/weights/vision_encoder_local.pt`
  - `hardware_engine/cognitive/weights/emg_encoder_local.pt`

### §6.2 Stage 2 · Fusion Head LOSO

```bash
python tools/train_fusion_head.py --exercise curl --epochs 50 --data-root data/v42
python tools/train_fusion_head.py --exercise squat --epochs 50 --data-root data/v42
```

- 冻结两 Encoder
- LOSO 3 折：(A+B→C, A+C→B, B+C→A)，每折训 90 + 验证 45
- 分类器：`Linear(21, 3) = 66 参数`
- 超参：Adam lr=1e-3, batch=16, dropout=0.3, weight_decay=1e-3, early-stop patience=5
- 期望：每折 ~2 分钟，弯举+深蹲共 ~12 分钟
- 通过标准：**三折 avg val_f1 ≥ 0.65，train_f1 − val_f1 ≤ 0.15**
- 产出：
  - `hardware_engine/cognitive/weights/v42_fusion_head_curl.pt` + `_metrics.json`
  - `hardware_engine/cognitive/weights/v42_fusion_head_squat.pt` + `_metrics.json`

`_metrics.json` 包含：三折 F1、混淆矩阵、cos_sim 95 百分位阈值（异常检测用）。

### §6.3 Stage 3 · 欠拟合救援（可选）

**仅当 Stage 2 avg F1 < 0.65 触发**：

```bash
python tools/train_fusion_head.py --exercise curl --deep --unfreeze-encoder --epochs 70
```

- `--deep`：扩分类器到 `Linear(21, 8) → ReLU → Linear(8, 3) = 203 参数`
- `--unfreeze-encoder`：解锁 Encoder 最后一层，lr=1e-4
- 仍然受参数预算 ≤ 200→203 红线（仅略微松动）

### §6.4 过拟合硬闸门

训练循环内自动计算 `train_f1 − val_f1`。若 > 0.15 立即终止并打印 warning。

**原理**：LOSO 每折只有 90 样本训练，21d 输入里有 16d 是 Encoder 抽取的 embedding，信息容量足以在小样本上过拟合。0.15 的 gap 是经验阈值，来自小样本学习理论：gap > 样本 std 通常意味着泛化失败。

出现过拟合时检查：
1. Dropout 是否生效（应 0.3）
2. Weight decay 是否生效（应 1e-3）
3. Early-stop patience 是否过大（应 ≤ 5）
4. 数据增强是否被正确应用

## §7 J1-J4 族群归属验证

族群归属判定的目的：证明本地硬件采到的 sEMG 属于主流 sEMG 信号族群，可以在答辩时声明复用 SENIAM / Ninapro / Camargo 的理论基线。基准值由 [`tools/compute_family_baselines.py`](../../tools/compute_family_baselines.py) 生成到 `docs/research/family_baselines.json`。

| 判据 | 指标 | 文献基准 | 通过阈值 |
|---|---|---|---|
| J1 频域落位 | 目标肌 MVC 时段 MDF | 弯举 87±15 Hz（Ninapro DB2）/ 深蹲由 Camargo 算 | \|Δ\| ≤ 20 Hz |
| J2 归一化幅值 | 中等负载时 `RMS/MVC` 分布 | 0.45±0.15 | 分布重叠 ≥ 50% |
| J3 SNR | 静息 RMS / MVC peak RMS | SENIAM < 1:10 | < 1:10 |
| J4 EMG-角度互相关 | 目标肌 RMS 包络 vs 关节角 | \|r\| > 0.7 | \|r\| > 0.6 |

## §8 无伪迁移审计

### §8.1 原则

**Ninapro / Camargo 只用于 L3**（噪声谱数据增强 + J1/J2 统计基准），**绝不**做：
- Encoder 预训练（cross-domain 不对齐，会污染）
- 权重初始化（数据集肌肉位置、电极型号、采样率都不同）
- 分类头迁移（标签体系完全不同，Ninapro 是静态手势，Camargo 是步态）

深蹲任何环节彻底**不用 Ninapro**（下肢无关）。

依据：velvet-ember plan §1 锁定决策 #5。

### §8.2 审计命令

```bash
# 期望：只在 tools/compute_family_baselines.py 的注释 / docstring 里出现
grep -rn "ninapro.*pretrain\|camargo.*pretrain" tools/

# 更严格版：检查 torch.load 调用上下文
grep -rn "torch.load" tools/ hardware_engine/cognitive/ | grep -iE "ninapro|camargo"
# 期望：无匹配
```

### §8.3 为什么伪迁移在本项目会失败

- **静态手势 ≠ 动态弯举**：Ninapro DB2 的肱二头肌数据是手势保持，无角度联动；弯举是连续等张收缩 + 关节角度相关联的 7D 张量。
- **sit-to-stand ≠ 深蹲**：Camargo 的 STS 深度只到 60° 膝角，深蹲到 90°；且 STS 无负重，深蹲可能有哑铃。
- **电极位置、采样率、ADC 位深、滤波带宽都不一致**：直接搬迁 Encoder 权重等于给模型喂噪声。

→ 预训练只用本地 270 rep 自监督 masked AE，是小样本下唯一正确的做法。
