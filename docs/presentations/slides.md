# IronBuddy V3.0：视感融合多模态居家 AI 健身教练
## 2026 ISEE-瑞芯微移动创新竞赛 最终答辩展示

---
## 第一章：项目概览与预期成果
- **核心定位**：基于边缘算力（NPU）与可穿戴计算的双核居家 AI 健身系统。
- **关键技术矩阵**：
  - **YOLOv5** 目标检测与人体实时骨骼捕捉。
  - 结合 **OpenClaw Agent** 框架底座与 **DeepSeek API** 本地推流。
- **硬件基石**：RK3399ProX 开发板 + ESP32 sEMG 阵列。
- **最终呈现**：实现零时差（<15m）、千帧级视觉与肌电（肌肉激活度度量）的高维重塑与自然交互闭环。

---
## 第二章：边缘测试接管（网络通信建立）
为了确保设备间的极低时延与高速下发，系统部署采用双端通讯模式：
- **第一阶（直连初始化）**：宿主机与 RK3399ProX 采用物理网卡直连，通过 Windows 网络共享模式建立最初的局域握手（如 `192.168.137.x`）。
- **第二阶（无线推流融合）**：通过 SSH 切入板内终端，利用 `nmcli`（或 `wpa_supplicant`）使得开发板直接挂靠在前端测试的“手机热点”下。电脑端也并入相同热点。
- **开发利器赋能**：在全套并行测试期间，极大地依靠 Agent 并发辅助调度与撰写了复杂的串联与排障策略。

## 第四章：NPU 神经网络引擎（YOLOv5 的取舍定局）
- **选型推理**：为什么在边缘弱算力设备上选用 YOLOv5 而不是更为庞大的 v7 或 v8 分支。主要聚焦于性能与预测帧率的极致 Trade-off。
- **数据集获取**：直接使用了现已成功训练过、且覆盖大多数运动受阻及关节粘连场景的数据集直接做一次阶段性跑通尝试。实现开箱即用的推理管线验证。

---
## 第五章：主系统核心轮回（状态机流水线）
- 视觉得到帧序列 -> 结合 ESP32 每隔 `1000Hz` 推送进内存的高速 `sEMG`/`IMU` 进行匹配 -> 对肌肉激活度指标（0-100% 包络）打分。
- 前后端严格通过 **双路并发独立渲染架构**：UI 甚至可以支持 *纯软打桩验证*。在 UDP 包遗失 > 150ms 触发*零遮挡降级防护吐司*（Toast 告警）。

---
## 第六章：大语言模型框架联通与拓展边界
- **OpenClaw 架构桥接**：利用 OpenClaw 建立的智能控制网关处理上下文流向。
- **DeepSeek API 集成逻辑**：使用低延迟策略向智能中心发送关键关节与指令流形。
- **语音控制与扩展模块**：完全离线的 VOSK 唤醒指令（“教练”），以及未来并线连接如飞书等更广互联体系以发送健身报告。

---
## 第七章：多模态总线联调（宏观大盘展示）
- **单板层**：YOLO 视频解析、Python 异步 `bleak` 的核心拉取与 NPU 并行算网调度。
- **云/边缘层**：挂靠的 AI 大模型推理逻辑与网络协同。
- **穿戴游离层**：随身运动小背包下的 ESP32 传感阵列无感采集。
一整套系统通过 Python 和底层共享内存（`/dev/shm`）形成一个极致低延迟的完美内聚循环。
