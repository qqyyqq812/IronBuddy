# 第十章：纯软解耦架构与容灾降级测试回溯 (Worker 1 & 2 实战阶段性总结)

本草稿章整合了在开发 Phase 3 阶段（多模态软硬件解耦冲刺期）前端展现与数据底座团队的测试结论。它是目前系统实现“千帧级零感阻塞”视觉核心的唯一技术备忘，也是答辩现场用于应对技术质询的终极“防坑底牌”。

## 1. 视-感流：双路高频并发渲染解耦架构
在实景评测下，传统绑定架构容易受 NPU 推理帧率波动（导致底层状态轮询迟滞）影响界面 UI（如红条呼吸跳动）。
* **破解方案**：完全剥离主循环。受限算力的慢速 FSM（如摄像头推流 `streamer_app`）在主线程运行，而前端用于描绘 EMG 肌肉爆发感的高频轮询（通过 `/api/muscle_activation` 达到 `100Hz`）在独立异步管线狂飙。
* **业务成效**：前端的呼吸红条顺滑度不再与 3D 小人模型的关键帧掉帧率发生耦合，成功轰出了如丝般顺滑的肌电响应感。

## 2. 系统弹压与保活：NPU 软打桩与优雅降级

### NPU 纯软打桩替身 (Mock Mode)
在无需拉升摄像头、不开扬声器、甚至 NPU 断电的环境（如教室内静默演示）下，系统已经全面支持纯软替代模式测试。只需通过 Python 脚本直插系统级内存共享区 `/dev/shm` 注入伪造的 JSON Pose 骨架点数据，即可完整全域激活 `index.html` 的 3D 人体深蹲反馈和包络线 UI。

### 优雅断连与降级（核心容灾点）
当传感器掉链子，或射频层遭遇强干扰时：
*   **现象与应对**：如果在 `150 毫秒`内底层通讯管线完全收不到心跳包，系统不会让核心进程死锁或卡死。取而代之的是，在展示大屏右上角弹出一个轻量化的轻便告警 Toast（`sensor-alert-toast`）。
*   **评委展示卖点**：即使传感器阵列失效，也不会遮挡 NPU 骨架的还原流程和摄像头的持续抓取工作。充分向评委展示了整个 V3.0 多模态微服务高内聚低耦合的健壮性（Robustness）。

## 3. 重大工程雷区预警（丢包序列号缺陷防范）
在 UDP 承载端实现了惊人的 `1000Hz` 肌电采样回压能力后，系统暴露出一个基于极客经验的数据包定义问题。

> [!WARNING]
> **底层丢包检测盲区！**
> 现行的 8-Byte UDP Payload 载荷协议虽然紧凑，但**缺乏核心单片机端自增序列号 (Sequence Number)** 作为包头校验位。这相当于在高压频次的丢包下，接收端完全无法侦测中间存在缺包现象。
> **在答辩现场若遇到评审关于高频射频传输（BLE/UDP）稳定性的挑剔质询，须严正指出：**该漏洞是在极端软硬件分离解耦测试时暴露出来的。团队计划在未来的 V4.0 更新中纳入 12-Byte 头尾校验闭环。这能反向秀出开发团队具备极其成熟的大规模网络吞吐排障实战素养。
