# 基于端云协同与双模态感知技术的大型智能嵌入式健身教练辅助系统 

## 第1章 项目设计目标与选题背景 (Introduction & Objective)
- 1.1 项目设计目标：打造一款低延时、具备生理与视觉感知能力的大型智能健身硬件系统。
- 1.2 现有技术局限与盲区：指出纯计算机视觉方案对深层肌群（肌肉不平衡、异常代偿）的监测盲区。
- 1.3 整体系统设计蓝图与创新点综述。

## 第2章 彻底的系统硬件集成设计 (Hardware Design & Ecosystem)
- 2.1 边缘侧传感集成硬件设计与图纸分析（展示由双路肌电传感器、ESP32核心板与独立电源模块所构成的高集成定制化原理图与PCB阵列）。
- 2.2 穿戴式物理人机工程学方案：摒弃冗杂有线束缚，采用“一体化主控腰包+分散式传感器贴片”设计，显著降低使用时的物理干涉。
- 2.3 无线网络通信及集线拓扑：ESP32 通过 Wi-Fi 网络下的 UDP 协议，将海量原始电信号以微秒级延迟高吞吐量透传至边缘计算主机（RK3399ProX）并建立个人热点局域网。同时搭配 ES7243 阵列音频收发。

## 第3章 核心软件算法与技术选型分析 (Methodology & Technical Routes)
- 3.1 面向嵌入式边缘算力的多模态模型演进：
    - 早期方案痛陈：分别对多模态源（纯图像、纯时序肌电波形）进行端到端重型神经网络跑批后再汇集，这不仅引发了模态脱节，更直接导致了 RK3399ProX 板端 NPU 的算力崩溃。
- 3.2 轻量级中继融合（Mid-Fusion）架构定型：
    - **视觉分流**：利用轻量化目标检测获取骨骼几何角度、运动相位及加速度；
    - **生理电分流**：引入频域特征提取与时域降维，剥离原始高频波。
    - **特征融合**：将两者预处理后的精炼结构化数据一并输入完全由本地自训练的极低参数量 GRU（门控循环单元）网段中提取非线性关系。
- 3.3 大语言模型（LLM）的智能协同重构：
    - 将原有粗放的一体化中层组件剥离；改由前端直接连入 DeepSeek API以获得 0 延迟语音级丝滑体验。
    - OpenClaw 退居幕后侦听器（Daemon），负责长期记忆存储清洗、本地 SQLite 数据发掘以及向外部系统（如飞书）的高级战报推送。
