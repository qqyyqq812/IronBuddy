# IronBuddy V3 多模态神经网络方案

> 目标指导：拒绝学术废话，注重落地逻辑与展示视觉冲击力。本方案统一呈现了 IronBuddy “视觉+肌电”跨模态打假的端到端实现路径与评委展示要点。

## 1. 竞赛大屏展示维度 (展示方案)
我们向评委展示的将是一个拥有“物理视觉透视”能力的严苛教练级系统。大屏呈现严格分割为三重维度：
1. **维度 1：动作达标物理计数**。只参考 NPU 视觉骨架，只判断“身体有没有按规定蹲下去/拉起来”，视觉界面计次数。
2. **维度 2：能量消耗/疲劳血条**。纯 DSP 数学处理：将目标肌肉通道的有效放电持续求积分，实时量化反应运动员肌肉储备“能量槽”的衰竭。
3. **维度 3：神经网络防作弊绝杀（核心护城河）**。在动作表象达标的情况下，通过微型融合网络抓取异常特征。一旦发现借力，大屏幕弹射高危报警框（如：“检测到下背部竖脊肌危险代偿发力，本轮深蹲无效！”），实现对传统纯摄像头的全维度降维打击。

## 2. 核心架构与 5D 特征输入层 (实现路径)
我们坚决**避开碰大视觉模型的内核**，也不写脆弱庞杂的所谓“if-else”逻辑。在主循环推流的最后，我们挂载一个**轻量级 GRU (门控循环单元)** 作为终裁法庭。

✅ **为何选定 GRU：** 动作有快有慢，借力代偿往往存在“错位的时间差”（先借力，后举起）。GRU 专门处理这种不确定长度的时序因果，不仅能敏锐捕捉“谁先发力谁后发力”的破绽，且运行代价在边缘板上不足单帧 2ms。

✅ **输入模型的扁平 5D 特征矩阵：**
在主状态机判定一个动作完整的瞬间，抽取该动作片段内的以下表格喂入模型：

| 特征项 | 来源类型 | 数据说明 |
| :--- | :--- | :--- |
| **时钟戳** (`Timestamp`) | 硬件对齐 | 强制弥合软硬件异构的时间隔阂 |
| **骨骼角速度** (`Ang_Vel`) | 视觉端派生 | 将视觉角度在时间轴上求导，敏锐抓捕“猛甩发力”等超模加速度 |
| **瞬时角度** (`Angle`) | 视觉基干 | 当前关键节点的几何位置 |
| **目标肌发力包络** (`Target_RMS`)| 传感器 DSP | **本该发力的肌肉读数**（判断是否出力） |
| **代偿肌发力包络** (`Comp_RMS`) | 传感器 DSP | **不该发力却发力爆表的位置读数**（抓取借力实锤证据） |

## 3. 实验动作库与贴片拓扑
为了训练出具备强说服力的打假网络，我们圈定以下两组经典测试场景布局：

### 场景 A：深蹲 (Squat) —— 狙击“膝腰杀手”
*   **目标通道 1**：臀大肌/股四头肌中心（正常深蹲的基石核发力点）。
*   **代偿通道 2**：下背部/竖脊肌（L1/L2脊椎水平外侧）。
*   **网络发力点**：抓住“蹲得很深，臀部却死水微澜，且下腰部肌电逆天飙升”的经典躯干前倾致命代偿。

### 场景 B：哑铃弯举 (Dumbbell Curl) —— 狙击“荡秋千”借力
*   **目标通道 1**：大臂前侧的肱二头肌。
*   **代偿通道 2**：肩膀前突隆起处（三角肌前束）或颈根（上斜方肌）。
*   **网络发力点**：抓住“由于太重，不由自主靠挺腰撅胯往后仰产生位移，导致三角肌异常爆发而二头肌发力滞后”的标准借力行为。

## 4. 训练集免标（Lazy-Labeling）极地工程
彻底废除“人工盯着几十万行 CSV 填 1 和 0”的蠢笨路线，采用 **Distant Supervision (数据集文件切割法)**：

1. **傻瓜式采集**：教练亲下场，录制连续 20 个教科书完美的极限标准深蹲，保存为 `golden_train.csv`。再发狂地录制 20 个全靠背部强行挑起的错误深蹲，保存为 `lazy_train.csv`。
2. **极速打标装载**：编写极地（本地宿主机） PyTorch 脚本，配置为：凡是读取带有 `golden` 前缀的表格，张量全自动化赋予 Label `0`；凡是读取 `lazy`，全部自赋权为 Label `1`。
3. **极品收敛**：几秒钟内完成模型训练并导出极轻的 `.pt` 或 `.onnx` 权重。直接挂载回 RK3399 生产环境！
