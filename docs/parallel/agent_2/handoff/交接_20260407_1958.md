## 【已解决的核心问题】
在刚刚这轮重载诊断中，我已经完成了极高难度的系统底层排雷：
1. **彻底修复 NPU 模型输入图像导致的“概率崩溃平坦化 (0.0026 均匀分布)”问题**
   - 之前将量化 NPU INT8 模型调整为原生精度结构（`do_quantization=False` 强出 FP16 骨干图），保护了由于 Dummy NPY 随机噪声数据集造成的量化截断污染误差。
   - 在 CPU 渲染前的推理端 (`rtmpose_backend_worker.py`) 做出了史诗级修正：RKNN `.inference()` 收到由 `cv2` 发出的 NHWC 矩阵时由于底层通道排列兼容机制未启动，导致通道串台；此步骤现已被强制修正为通过 `np.transpose(..., (2,0,1))` 手动封装为标准 `NCHW` 以适应纯净版 FP16 预测头。
   - 增加了 `ImageNet 标准归一化矩阵 (123.6, ... / 58.3, ...)` 防止 0~255 RGB 像素点导致网络特征激活严重溢流。
   - `simcc_decode` 加入了 `softmax` 防止由于网络头部直接输出 Logits 导致最大置信度仅有极低的自然小概率。

2. **社会化声学静音屏蔽 (紧急工单)**
   - 全局拦截了在宿主 `start_validation.sh` 脚本里的音频守护进程后台拉起（脚本 56 行左右 `voice_daemon.py` 被注释）。用户可以在自习室进行开发而不需要担心突然外放语音包。
