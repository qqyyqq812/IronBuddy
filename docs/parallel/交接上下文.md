# IronBuddy V3 核心交接上下文 (The Grand Context)

> **这是一份全局事实备忘录。所有 Agent 在构思任何一行代码前，必须先吃透此文件。**

## 1. 我们的项目背景是什么？
IronBuddy V3 是一套部署在 **RK3399ProX NPU 边缘板卡** 上的高性能多模态体测控制台。它可以利用相机看到你在做深蹲，并通过局域网高速 UDP 读取挂载在你大腿上的 ESP32-ADC 原生表面肌电信号（sEMG）。

## 2. 现有的核心成果与物理定律（不可篡改）
- **高速异步前端**：主渲染页面在 `templates/index.html`。之前我们成功通过在 fetch 时追加 `location.origin` 以及强去重方案解开了 Flask 导致的前端死锁。推流程序叫 `streamer_app.py`。
- **肌电读取网关**：ESP32 以每秒 1000次 狂暴抛出浮点，由板载 `hardware_engine/sensor/udp_emg_server.py` 切割、去噪并提取出激活数值抛到 `/dev/shm/muscle_activation.json`。
- **主骨架裁定**：动作的计算与惩罚大脑位于 `hardware_engine/main_claw_loop.py` 中的 `SquatStateMachine`。
- 上述架构已历经数百次痛苦迭代，**严禁为了贪图便利去整体推翻这几大中枢引擎**，只能基于现有架构进行微创优化与挂载。

- 永远**不要向宿主机打印疯狂滚屏的 `print()`**，所有重要 Log 走 `/dev/shm` 或者专门的异步 logger。
