# 任务卷宗：Agent 1 — 语音通讯链路部
> 你负责底层 Voice Daemon 麦克风阵列与 ASR/TTS 对接通道的绝对顺畅。

## 你的行为铁律
1. **一次对话只做一个里程碑。** 开工前先出 implan，用户审批通过后才动手。
2. **完成里程碑后：** 执行 `/para-sync` 更新 status.md，在 drafts/ 下沉淀产出文件。
3. **对话结束前：** 无论是否完成，必须同步 status.md 并更新 handoff/ 交接记录。

## 本轮里程碑
- **节点 1**：解除麦克风在后台死锁与阻塞现象 (验收标准：在测试页点击语音唤醒不会全网瘫痪)。
- **节点 2**：稳定向 `main_claw_loop.py` 喂入正确的识别文字 (验收标准：DeepSeek 大语言模型有响应)。

## 关键技术上下文
> ⚠️ 你的核心战区在 `hardware_engine/voice_daemon.py`
我们系统当前的瓶颈是录音设备被硬件级阻塞（死锁），直接导致整个对话教练系统“失聪”。你必须绕过传统的 ALSA 线程堵塞，必要情况可以改写底层的 `arecord` 流管道来非阻塞地提取音频。一旦拿到音频，必须无感地传递给 `/dev/shm` 或者进程间通信信道，保障大模型代理获取真实指令。
