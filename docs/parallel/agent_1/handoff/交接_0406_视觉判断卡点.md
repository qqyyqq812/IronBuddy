# 交接文档：里程碑四_前端死锁与视觉判定卡点
**日期**：2026-04-06

## 1. 核心决策与成果
1. **彻底解放了发音器禁锢**：移除了 `voice_daemon.py` 中的 `mpg123 -f 200` 强行限制音量代码以及强降 `1%` 的 amixer 指令。现在语音回答极度洪亮。
2. **大模型延迟消除**：把 `main_claw_loop.py` 内部发送给 DeepSeek 的提示词强制增加 “不用通过 `<think>` 思考” 的铁律，消除了大模型前置的空转与超时等待。
3. **按键防抖与盲区修复**：前端增加了 `document.disabled=true` 及 UI Loading 动效，且修正了 `STATE_MAP` 字典缺失 `STAND` 状态导致假死的 bug。

## 2. 踩过的坑与遗留阻塞
- **终极噩梦：状态机永远不进状态**
  - **现象**：虽然能够完美跑通 `/reset_session` (FSM计数清理无误)，并且 MJPEG 视频流中也看到了绿色的边界框和关节坐标点（不仅能看清甚至画面很流畅），但**深蹲计数岿然不动**。
  - **深层剖析**：我们已经放宽了角度判定（`110`算蹲稳，`145`算站起），且尝试加入了基于 RTMPose 关键点 `Score` 的左右腿动态匹配——但实机运行中依然毫无反应（前端要么停在 `180°` 要么就因为姿态判定无法跌破下蹲阈值而不触发计数）。
  - **结论**：现存的一维（单纯连线算三角函数）纯视觉判定阈值极其脆弱！容易受到摄像头焦距、用户站位与身体畸变的严重降维打击。

## 3. 下次继承者的起点
当你（或下一次唤醒本 Agent 实例时）接手该项目，请：
1. **不要再徒劳地微调角度 Threshold 阈值参数了**！
2. 建议直接在 `SquatStateMachine.update()` 里打印出被捕捉到的那一侧 hip, knee, ankle 的真实计算角度（可以直接向 NPU 的 log 里注入），看看人即使蹲下来了，神经网络在那个视角下到底折算出的是多少度！
3. （更推荐）将视角计数问题作为遗留给后续接盘的 **Agent 3 (肌电-视觉融合神经网络)** 来做维度降维打击，早日逃离这个拼凑出来的条件状态机。
