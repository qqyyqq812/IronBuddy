## 测试 C: 状态机 + 计数

### C1: 深蹲检测
1. 站在摄像头前，做标准深蹲 (膝盖角度 < 110 度)
2. **预期**: 网页 "标准深蹲" 计数 +1，HUD 状态显示 "下蹲中"
3. **预期**: 疲劳值增加
4. **结果**: [ ] 通过 / [ 1] 失败 — 备注:现在遇到的情况是，之前还能够有正常的API. I输出，现在的话，他做满10做满1500疲劳之后，他就不输出了，可能是API掉线了帮我检查一下原因

### C2: 违规检测
1. 做半蹲 (角度 > 110 度就起来)
2. **预期**: "违规半蹲" 计数 +1
3. **预期**: 音箱发出警告声 (蜂鸣或语音)
4. **结果**: [ ] 通过 / [ 1] 失败 — 备注:这里有一个很大的状态机问题，就是它不能像以前我写好的那种可以非常清楚的判断出90°左右，90°100°以下吧就是标准深蹲，然后不到100°蹈就是不标准深蹲。它应该分为正确的这个状态机过程，但是现在我呃几乎所有的动作都会被认定为标准，并且呃做不标准的时候小音响是不会响的

### C3: 重置统计
