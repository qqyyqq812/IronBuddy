# IronBuddy V3.0 — 肌电传感器选型说明 (最终定案)

> 📅 2026-03-28 | 状态：最终敲定

## 1. 最终选型结论
**选型方案**：MyoWare 2.0 Muscle Sensor + Wireless Shield (全程无线方案)
**节点数量**：2 个独立节点（覆盖核心竞赛演示动作的两个肌群：如二头肌 + 股四头肌）
**预算总计**：约 ¥1035

## 2. 硬件架构
- **数据采集**：MyoWare 2.0 Muscle Sensor（3电极差分，输出模拟信号）
- **信号处理与传输**：MyoWare 2.0 Wireless Shield（板载 ESP32-WROOM，读取模拟量并作为 BLE Peripheral 广播）
- **供电**：Wireless Shield 内置 40mAh 聚合物锂电池
- **接收端**：RK3399ProX 板载 BT4.2 模块（不使用外接扩展板）

## 3. 通信链路
1. 传感器节点 (ESP32) 烧录 [官方 SparkFun 固件](https://github.com/sparkfun/SparkFun_MyoWare_Arduino_Library)，开启 BLE Peripheral 模式，暴露 Notify 特征。
2. 配置 2 个节点的 `localName` 为 `MyoWare-BICEP` 和 `MyoWare-QUAD`。
3. RK3399ProX 运行 Python 守护进程，利用 Linux BlueZ + `bleak` 库并发连接 2 个 BLE 节点。
4. RK3399ProX 解析后写入内存盘 `/dev/shm/sensor.json` 供主状态机（FSM）调用。

## 4. 选型优势
* **零导线束缚**：每个肌肉贴片完全独立自主工作，免去了长导线的拉扯，答辩演示自由度拉满。
* **高性价比**：与数万元甚至上十万元的国际专业科研装备（如 Delsys，Noraxon）相比，不到千元的低成本完成了完全无线的原型验证。
* **开源生态良好**：SparkFun 提供成熟开源 Arduino 库支持，二次开发风险极低。
