## 【已解决的核心问题】
本窗口成功定点并修复了 RKNN NPU 部署流程中经典的 “无报错但全抗毁” (特征图纯垃圾化) 现象：
1. **输入通道错乱与排布修正 (NHWC -> NCHW)**：RKNN 底层 Python API 吞吐时只校验显存块尺寸边界 (`192x256x3 = 147456`)，默认沿用了 ONNX 模型要求的 NCHW，而此前的 `cv2.resize` 送入了原生的 NHWC `[256, 192, 3]` 数组。通道交错切片导致 NPU 其实看到的全是高频噪点。已在 `rtmpose_backend_worker.py` 的 `preprocess` 中新增了 `np.transpose` 实施排布纠正。
2. **硬件 ImageNet 图像缩放丢失 (FP16 特性)**：我们废弃了之前强行压平了分布张量的毁坏性 INT8 随机噪声量化构建，成功修改 `convert_rtmpose.py` 输出纯净高精度 FP16 引擎拓扑。但因为 RKNN 抛去了 INT8 后不再自包均值缩放，必须由我们在 CPU 端手动使用 ImageNet 归一化 (减去 `[123.6, 116.2]`, 除以 `[58.3, 57.1]`) 进行预洗涤。此工作我也已写入 `preprocess` 中。
3. **彻底静音与拔除播报子系统**：板子环境音已强制用 `amixer sset Master mute` 抹除并直接中止了守护进程。宿主机根目录中调度的 `start_validation.sh` **第 54 行已被我严密封锁注释**，未来任何使用 `./start_validation.sh` 拉起的作业绝对不会再弹响 `voice_daemon.py`。
