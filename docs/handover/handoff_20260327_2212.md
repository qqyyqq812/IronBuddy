## 【已解决的核心问题】

1. **项目架构大一统重组**：彻底废除了早期 Agent1 (PC端) / Agent2 (板端) 的割裂并行探索架构，删除了全部 `agent/`, `agent_memory/`, `src/`, Agent 分区文档，回归以 `hardware_engine/` 为单一收口的工业级标准目录。
2. **竞赛级展示方案定案**：确立了"视觉算坐标(YOLO+3D Lifting) → IMU 抗遮挡纠偏 → EMG 肌肉红光热力渲染"的三层融合展示架构，以及"分屏对照法 + 现场拔插测试"的不可伪造证据链。
3. **硬件选型收敛**：从美诺韦医疗、ESP32+MPU6050 DIY、OYMotion gForce 三条路线中，收敛至 gForcePro+（8通道EMG+9轴IMU一体化弹力臂环，80g，5秒穿脱，开源 SDK），解决了"贴片太多太复杂"的致命穿戴性问题。
4. **与 VR 健身的差异化定位**：明确了"VR 是游戏（只追踪手柄位置），IronBuddy 测真实肌电（运动科学级）"的答辩话术。

- "有没有必要再加这个加速度传感器，这两个会不会重复"（答：不重复，EMG管发力/IMU管空间，且gForce臂环已自带两者一体化）

### 核心红线约束
1. **展示导向 (Presentation-Driven)**：一切技术决策以"答辩评委能否被震撼"为终极判据。
2. **传感器数据不进神经网络**：EMG/IMU 走轻量 RMS 特征工程 → FSM 规则融合，不训练多模态模型。
3. **前端为手机原生 APP**（而非微信小程序）。
4. **JSON 契约解耦**：队长与队友通过 `/dev/shm/sensor.json` 物理隔离，各自独立开发。
5. **禁止乱扔垃圾**：遵循七大基石制，所有文件必须在规范位置。
6. **中文 Markdown 规范**：所有文档以中文撰写。
