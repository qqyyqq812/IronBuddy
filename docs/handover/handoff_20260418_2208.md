# 交接文档 (Handoff) - 20260418_2208

## 板块 1：项目全景介绍
**IronBuddy 智能健身教练系统** 是一套位于边缘计算板端的物理监控平台系统。通过硬件采集肌电信号(EMG)并利用广角摄像头执行动作位姿识别，结合 FSM（状态机）研判用户的健身动作（“深蹲”、“弯举”）是否违规。对于违规行为，它不仅本地大叫，还能调用云端版 DeepSeek 输出评语并将其分发外围渠道。

**核心文件结构：**
- `/hardware_engine/voice_daemon.py`：负责 ASR/TTS 麦克风窃听与唤醒匹配（含 L0 强警报抢占逻辑），并挂载飞书关键字识别。
- `/streamer_app.py`：Flask 后端总控枢纽，承担对外的 API 注册（如 `/api/feishu/send_plan`）、前端调配。
- `/templates/index.html`：项目控制台与交互前端（原生 HTML + JS），不含现代框架，已在近几个版本被大幅瘦身处理。
- `docs/` 和 `brain/` 等作为辅助存档基类（含各种知识和快照）。

**技术栈与环境：**
Python, Flask, Baidu ASR/TTS API, DeepSeek API, Vanilla JS+HTML。部署环境为 Linux 板端，以共享内存（`/dev/shm/*`）实现底层进程互斥与通信（如 `mute_signal`, `fsm_state.json`）。
